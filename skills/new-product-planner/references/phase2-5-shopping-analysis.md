# PHASE 2.5 — 네이버 쇼핑 데이터 분석 가이드

## 개요

네이버 쇼핑 검색 API + 데이터랩 쇼핑인사이트 API를 활용하여 신제품의 시장 경쟁력과 성공 가능성을 데이터로 검증한다. PHASE 2 시장 조사와 PHASE 3 경쟁사 분석 사이에 실행하며, 두 단계의 근거를 데이터로 강화한다.

## 필요한 인증 정보

네이버 개발자센터(developers.naver.com) 애플리케이션의 인증 정보 (검색광고 API와 별도):
- `NAVER_CLIENT_ID`: 네이버 개발자센터 Client ID
- `NAVER_CLIENT_SECRET`: 네이버 개발자센터 Client Secret

사용자에게 안내: "네이버 쇼핑 데이터 분석을 위해 네이버 개발자센터(developers.naver.com)의 Client ID와 Secret이 필요해요. 검색광고 API 키와는 별도입니다."

## 5가지 분석 기능

### 분석 1: 경쟁사 제품 자동 수집

쇼핑 검색 API로 키워드 검색 → 상위 100개 상품의 이름, 가격, 브랜드, 이미지, 구매 링크를 한꺼번에 수집.

활용:
- PHASE 3 경쟁사 분석의 입력 데이터로 활용
- 리포트의 경쟁사 탭에 제품 링크+이미지 자동 삽입
- 수작업 조사 시간 대폭 절약

실행: `scripts/naver_shopping_analysis.py search --query "아기 쿨매트" --total 100`

### 분석 2: 가격대 분석

수집된 상품의 최저가 데이터로 시장 가격 분포를 파악.

산출물:
- 최저/최고/평균/중앙 가격
- 가격대별 분포 (1만원 미만 / 1~3만 / 3~5만 / 5~7만 / 7~10만 / 10만+ 비율)
- 프리미엄 구간 기준선 (상위 25% 가격)
- **가격 포지셔닝 추천**: "가성비 / 주류 / 프리미엄" 3구간 제시

활용:
- 신제품 가격 책정의 데이터 근거
- "프리미엄 포지셔닝 시 X~Y원이 적정" 형태로 사업 타당성 섹션에 반영

### 분석 3: 시즌별 수요 트렌드

데이터랩 쇼핑인사이트 API로 해당 카테고리의 월별 검색 클릭 추이를 조회.

산출물:
- 12개월 이상의 월별 검색량 추이 그래프
- 성수기/비수기 자동 식별
- 전년 동기 대비 성장/감소 트렌드

활용:
- 출시 타이밍 최적화 (성수기 2~3개월 전 준비 시작)
- 마케팅 예산 배분 근거 (성수기에 집중)
- 재고 계획 (수요 예측)

### 분석 4: 타겟 고객 검증

데이터랩에서 성별/연령별 검색 클릭 비율을 조회하여 실제 타겟 고객을 데이터로 검증.

산출물:
- 성별 비율 (예: 여성 78%, 남성 22%)
- 연령대별 비율 (예: 30대 45%, 20대 25%, 40대 20%)
- PC vs 모바일 비율

활용:
- "우리 타겟이 30대 여성인데, 실제 데이터에서도 30대 여성이 주 검색층" → 타겟 정합성 확인
- 마케팅 채널 선정 근거 (모바일 비중 높으면 인스타/맘카페 우선)

### 분석 5: 브랜드 점유율 추정

쇼핑 검색 상위 100개 결과에서 브랜드별 등장 빈도를 계산.

산출물:
- 브랜드별 검색 결과 점유율 TOP 15
- 판매 쇼핑몰별 분포 TOP 10
- 시장 집중도 판단 (상위 3개 브랜드 합산 점유율)

활용:
- 시장 집중도가 낮으면 → 진입 기회 크다
- 특정 브랜드가 독점이면 → 차별화 전략 필수
- 소규모 브랜드가 많으면 → 브랜드 파워로 승부 가능

## 카테고리 ID 찾는 방법

데이터랩 API는 카테고리 ID가 필요하다. 찾는 방법:
1. 네이버 쇼핑 (shopping.naver.com) 접속
2. 원하는 카테고리로 이동
3. URL에서 `catId` 파라미터 확인
   예: `https://shopping.naver.com/...?catId=50000008` → ID는 `50000008`

또는 `web_search`로 "네이버 쇼핑 카테고리 ID [카테고리명]" 검색.

주요 유아용품 카테고리 ID 참고:
- 출산/유아동: 50000008
- 유아동 의류: 50000009
- (하위 카테고리는 네이버 쇼핑에서 직접 확인 필요)

## 실행 스크립트

`scripts/naver_shopping_analysis.py` 3가지 모드:

```bash
# 쇼핑 검색만 (분석 1,2,5)
python naver_shopping_analysis.py search \
  --client-id "YOUR_ID" --client-secret "YOUR_SECRET" \
  --query "아기 쿨매트" --total 100

# 트렌드만 (분석 3,4)
python naver_shopping_analysis.py trend \
  --client-id "YOUR_ID" --client-secret "YOUR_SECRET" \
  --category "50000008" --category-name "출산유아동" \
  --keywords "아기 쿨매트,아기 냉감패드" \
  --start-date "2025-01-01" --end-date "2026-03-01"

# 전체 통합 (분석 1~5 전부)
python naver_shopping_analysis.py full \
  --client-id "YOUR_ID" --client-secret "YOUR_SECRET" \
  --query "아기 쿨매트" --category "50000008" \
  --category-name "출산유아동" \
  --keywords "아기 쿨매트,아기 냉감패드,아기 여름이불"
```

## Claude 환경에서의 대안

네이버 API 도메인(`openapi.naver.com`)이 차단된 경우:
1. web_search로 네이버 쇼핑 검색 결과 페이지를 수집하여 유사 데이터 추출
2. 네이버 데이터랩 웹 화면의 공개 데이터를 참조
3. 사용자에게 로컬 PC 실행 후 JSON 업로드를 안내

## 리포트 통합

분석 결과는 HTML 리포트의 다음 탭에 반영:
- **시장 분석 탭**: 시즌 트렌드 차트 + 타겟 고객 검증 데이터
- **경쟁사 분석 탭**: 자동 수집된 경쟁 제품 링크/이미지 + 브랜드 점유율 차트
- **사업 타당성 탭**: 가격대 분석 → 가격 포지셔닝 추천 + 시장 기회 데이터 근거
