# PHASE 6 — 제품 컨셉 확정 & HTML 리포트 생성 가이드

## 최종 컨셉 확정

PHASE 1~5의 모든 결과를 종합하여 사용자와 함께 최종 제품 컨셉을 확정한다.

### 확정 체크리스트

사용자에게 아래 항목을 하나씩 확인:
- [ ] 제품명 (가칭이라도 OK)
- [ ] 한 줄 컨셉 확정
- [ ] 타겟 고객 명확화
- [ ] 핵심 USP 3개 확정
- [ ] 예상 가격대
- [ ] 출시 목표 시기
- [ ] 핵심 마케팅 키워드 TOP 10

"이 내용으로 최종 기획 리포트를 만들까요? 수정할 부분 있으면 말씀해주세요."

## 인터랙티브 HTML 리포트 구조

최종 산출물은 탭 기반 인터랙티브 HTML 리포트.
아래 구조와 디자인 가이드를 따라 생성한다.

### 전체 레이아웃

```
┌─────────────────────────────────────────────┐
│  📋 [브랜드명] 신제품 기획 리포트            │
│  [제품명 (가칭)] — [날짜]                    │
├─────────────────────────────────────────────┤
│ [컨셉] [시장] [경쟁사] [키워드] [검색량] [로드맵] │
├─────────────────────────────────────────────┤
│                                             │
│  (탭 내용 영역)                               │
│                                             │
└─────────────────────────────────────────────┘
```

### 탭 1: 제품 컨셉 요약

| 항목 | 내용 |
|------|------|
| 제품명 (가칭) | |
| 한 줄 컨셉 | |
| 타겟 고객 | |
| 해결하는 문제 | |
| 핵심 USP | 1. / 2. / 3. |
| 예상 가격대 | |
| 출시 시기 | |
| 차별화 포인트 | |
| 리스크 요인 | |

### 탭 1: 제품 컨셉 요약 (기존 + 레퍼런스 이미지)

상단: 핵심 컨셉 정보 카드 그리드
하단: 대표 레퍼런스 이미지 2~3장 (갤러리에서 하이라이트)
+ 아이디어 발전 과정 요약 (PHASE 1 → 최종 컨셉)

### 탭 2: 사업 타당성 (신규 — 핵심 섹션)

→ 세부 가이드: `references/phase6-business-justification.md`

상단 카드:
| 시장 기회 | 브랜드 적합성 | 매출 갭 해소 | 경쟁 우위 |

본문:
- 시장 기회 요약 + 데이터 근거
- 브랜드 시너지 분석
- 매출 갭 분석 (비수기 → 신제품으로 채울 수 있는 규모)
- 리스크 & 대응 테이블 (리스크 / 심각도 / 대응방안)
- 예상 ROI 시나리오 (보수적 / 기본 / 낙관적)

### 탭 3: 시장 분석

- 시장 규모 & 성장률 (가능하면 차트)
- 핵심 트렌드 3~5개 (카드 형태)
- 소비자 니즈 & 페인포인트 (리스트)
- 계절성 패턴 (있다면)
- 시장 기회 요약

### 탭 4: 경쟁사 분석 (업그레이드 — 링크 포함)

- 경쟁사 비교표 (테이블, 정렬 가능) + **각 경쟁사 제품 URL 클릭 링크**
- 포지셔닝 맵 (2x2 매트릭스 시각화)
- 각 경쟁사 강점/약점 요약
- 우리의 차별화 포지션

### 탭 5: 레퍼런스 보드 (신규 — 이미지 갤러리)

→ 세부 가이드: `references/phase3-reference-images.md`

- 카테고리별 필터: 경쟁사 제품 / 해외 레퍼런스 / 디자인 무드 / 사용자 업로드
- 이미지 그리드 (3열): 썸네일 + 설명 + 출처 링크
- 자동 수집(image_search) + 사용자 업로드 이미지 통합 표시

### 탭 6: 키워드 분석

- 전체 키워드 리스트 (테이블, 필터/정렬 가능)
  - 컬럼: 키워드 / 검색의도 / 경쟁도 / 추천채널 / 우선순위
- 검색의도별 분포 차트 (도넛 차트)
- 키워드 우선순위 매트릭스

### 탭 7: 네이버 검색량

- 키워드별 검색량 바 차트 (상위 20개)
- 등급별 분류 (골든/성장/빅/니치 색상 구분)
- 상세 데이터 테이블 (PC/모바일 검색량, 경쟁도, 클릭률)
  - 필터: 등급별 필터링 가능
  - 정렬: 각 컬럼 클릭 시 정렬
- 모바일 비중 분석
- 연관 키워드 발견 (API에서 추가로 나온 것들)

### 탭 8: 실행 로드맵

- 다음 단계 액션 아이템 (타임라인 형태)
- 우선순위별 할 일 목록
- 예상 일정
- 필요 리소스

## HTML 디자인 가이드

### 색상 팔레트
- 메인: #2563EB (파란색 계열)
- 서브: #7C3AED (보라색 계열)
- 성공: #10B981
- 경고: #F59E0B
- 위험: #EF4444
- 배경: #F8FAFC
- 카드 배경: #FFFFFF

### 등급별 색상
- 골든: #F59E0B (금색)
- 성장: #10B981 (녹색)
- 빅: #EF4444 (빨간색)
- 니치: #6B7280 (회색)

### 컴포넌트 스타일
- 카드: border-radius 12px, box-shadow 미세하게
- 테이블: 줄무늬, hover 효과, 헤더 고정
- 차트: Chart.js 또는 순수 CSS/SVG 바 차트
- 탭: 하단 밑줄 스타일, 활성 탭 강조

### 반응형
- 데스크톱 (1200px+): 2~3컬럼 그리드
- 태블릿 (768px~): 2컬럼
- 모바일 (768px 미만): 1컬럼, 테이블 가로 스크롤

## 리포트 생성 실행

HTML 파일 생성 시:
1. 모든 데이터를 JavaScript 변수로 임베드
2. 탭 전환은 순수 JavaScript (라이브러리 없이)
3. 테이블 정렬/필터도 순수 JavaScript
4. 차트는 인라인 SVG 또는 CSS 바 차트 (외부 의존성 최소화)
5. 단일 .html 파일로 완결 (오프라인에서도 열 수 있게)
6. 최종 파일을 `/mnt/user-data/outputs/` 에 저장하고 `present_files`로 제공
