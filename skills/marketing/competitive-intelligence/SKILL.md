# 경쟁사 인텔리전스 v2

## 브랜드 컨텍스트

- **브랜드**: 썬데이허그 (SundayHug) / 주식회사 제이코프 (JAYCORP)
- **카테고리**: 아기 수면 용품 (슬리핑백, 속싸개, 접이식 침대, 백색소음기, 역류방지쿠션)
- **가격대**: 30,000~150,000원대 프리미엄 포지셔닝
- **주요 채널**: 네이버 스마트스토어, 쿠팡, 자사몰(sundayhug.kr)
- **타겟**: 0~5세 자녀를 둔 30대 부모

## ⚡ 모드 A — 단일 경쟁사 심층분석 (온디맨드)

**트리거**: 특정 브랜드명이 언급되거나 "OOO 분석해줘" 요청

### 입력
- 경쟁사 브랜드명 (필수)
- 네이버 스토어 URL 또는 자사몰 URL (선택 — 없으면 자동 검색)

### 실행 플로우 (6단계)

#### STEP A1: 브랜드 탐색 & 채널 매핑

브랜드명으로 아래 채널을 자동 탐색:

```
1. web_search: "{브랜드명} 네이버 스마트스토어"
2. web_search: "{브랜드명} 쿠팡 스토어"
3. web_search: "{브랜드명} 공식 홈페이지"
4. web_search: "{브랜드명} 인스타그램"
```

결과로 채널 맵 생성:
- 네이버 스마트스토어 URL
- 쿠팡 스토어 URL
- 자사몰 URL
- 인스타그램 계정

또한 `references/known-competitors.md`에서 기존 정보 확인 후 보강.

#### STEP A2: 제품 라인업 & 가격 수집

**네이버 스마트스토어 스크래핑**:

`web_fetch`로 스토어 상품 목록 페이지를 수집하거나, `scripts/naver_scraper.py` 실행:
```bash
python3 /path/to/scripts/naver_scraper.py --mode products --query "{브랜드명}"
```

**수집 항목**:
| 항목 | 방법 |
|------|------|
| 전체 상품 수 | 스토어 홈 / 네이버쇼핑 검색 |
| 상품별 가격 | 상품 목록 |
| 할인/쿠폰 정보 | 상품 상세 |
| 무료배송 여부 | 상품 상세 |
| 리뷰 수/평점 | 상품 목록 |
| 베스트셀러 | 리뷰수 기준 TOP 10 |

**가격 분석**:
- 최저/최고/평균/중앙값
- 가격대 분포 (1만원 구간)
- 썬데이허그 동일 카테고리 제품 대비 포지셔닝

#### STEP A3: 프로모션 & 행사 분석

수집 대상:
```
1. web_search: "{브랜드명} 할인 이벤트"
2. web_search: "{브랜드명} 체험단"
3. web_search: "{브랜드명} 공동구매"
4. web_search: "{브랜드명} 기획전"
5. web_fetch: 네이버 스토어 홈 배너 (URL 확보된 경우)
```

감지 키워드:
```
할인, 세일, SALE, 특가, 무료배송, 사은품, 증정, 1+1, 공구, 
체험단, 공동구매, 기획전, 한정, 쿠폰, 적립금, 포인트
```

출력: 현재 진행 중인 프로모션 목록 + 유형 분류

#### STEP A4: 마케팅 채널 분석

**인스타그램** (공개 계정):
```
1. web_search: "{브랜드명} 인스타그램"
2. web_fetch: 인스타 프로필 페이지 (가능한 경우)
3. web_search: "site:instagram.com {브랜드명}"
```
분석 항목:
- 팔로워 수 (근사치)
- 최근 게시물 빈도
- 콘텐츠 유형: 제품컷 / 라이프스타일 / UGC / 이벤트 / 릴스
- 주요 해시태그 TOP 15
- 협찬/광고 표시 비율

**네이버 마케팅**:
```
1. web_search: "{브랜드명} site:blog.naver.com" → 체험단 규모 추정
2. web_search: "{브랜드명} 블로그 체험단 모집"
3. web_search: "네이버 쇼핑 {브랜드명}" → 광고 집행 여부 확인
```

**콘텐츠 전략 요약**:
- 주력 소구 메시지 (USP 키워드)
- 비주얼 스타일
- CTA 유형

#### STEP A5: 리뷰 & VOC 분석

**리뷰 수집**:
```
1. web_search: "{브랜드명} 후기"
2. web_search: "{브랜드명} 리뷰 네이버"
3. web_search: "{브랜드명} 후기 site:cafe.naver.com"
4. web_search: "{브랜드명} 솔직후기"
5. web_search: "{브랜드명} vs 썬데이허그"
```

가능하면 네이버 쇼핑 리뷰 페이지를 `web_fetch`로 직접 수집.

**감성 분석 키워드**:
```
긍정: 따뜻해요, 잘자요, 편해요, 추천, 재구매, 부드러워요, 고급, 디자인좋아요
부정: 비싸요, 불편, 사이즈, 세탁, 배송, 품질, 환불, 올풀림, 땀
중립: 어때요, 비교, 궁금, 보통
```

**VOC 핵심 도출**:
- 장점 TOP 5 / 불만 TOP 5
- 경쟁사 리뷰에서 발견되는 썬데이허그의 기회

#### STEP A6: 인터랙티브 HTML 리포트 생성

→ 상세 스펙: `references/single-brand-spec.md`

보고서를 `scripts/report_builder.py`로 생성하거나, 직접 HTML을 작성.

**HTML 리포트 구조 (탭 7개)**:
1. 📊 **프로필 요약** — 브랜드 개요, 채널맵, 핵심 수치
2. 🛒 **제품 & 가격** — 라인업 테이블, 가격분포 차트, 베스트셀러
3. 🏷️ **프로모션** — 진행 중 행사, 체험단/공구 현황
4. 📱 **마케팅 분석** — 인스타/블로그/검색 마케팅 전략
5. ⭐ **리뷰 & VOC** — 감성분석 차트, 장점/불만 워드클라우드
6. ⚔️ **vs 썬데이허그** — 가격/리뷰/마케팅 직접 비교
7. 💡 **인사이트** — 배울 점, 기회, 대응 액션 아이템

파일 저장: `~/Desktop/team-skills/리포트/ci_{브랜드명}_{YYYYMMDD}.html`

**리포트 디자인**:
- 단일 HTML 파일 (외부 의존성 없음)
- 다크/라이트 테마 없이 클린 화이트 배경
- CSS 변수: `references/single-brand-spec.md`의 디자인 시스템 참조
- Chart.js CDN (`https://cdnjs.cloudflare.com/ajax/libs/Chart.js/4.4.0/chart.umd.min.js`) 사용 가능
- 반응형 (모바일 대응)

## 주의사항

- 네이버/쿠팡 크롤링은 `web_fetch` 기반, 과도한 요청 자제
- 인스타그램은 공개 계정/게시물만 수집
- 맘카페는 공개 게시물만, 개인정보 제외
- 크롤링 실패 시 → web_search 폴백으로 대체 데이터 수집
- 데이터 수집 불가 항목은 "데이터 없음"으로 표시, 거짓 데이터 생성 금지
- HTML 리포트는 단일 파일 (외부 의존성 없음, CDN은 허용)
