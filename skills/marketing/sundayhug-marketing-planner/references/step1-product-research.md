# STEP 1 — 제품 정보 수집 가이드

## 수집 방법

### A. URL 제공 시 (우선)
```
web_fetch(url) 실행
→ 상세페이지 크롤링
→ 아래 항목 추출
```

추출 항목:
- 제품명 / 카테고리
- 판매가 / 할인가
- 소재 (혼방 비율 포함)
- 사이즈 구성 (S/M/L 등)
- 핵심 기능 (투웨이 지퍼, 낙상방지 등)
- 컬러/패턴 옵션
- 추천 온도/시즌
- 실제 리뷰 텍스트 (최소 5개)

### B. 정보 직접 입력 시
사용자에게 아래를 질문:
1. 제품명과 카테고리는?
2. 소재(원단)는? (밤부/메쉬/트리플 등)
3. 주요 기능/차별점은?
4. 타겟 시즌(여름/겨울/사계절)?
5. 가격대는?

## 리뷰에서 키워드 채굴하기

실제 구매자 리뷰에서 다음 패턴을 찾는다:

**고통 포인트 언어** (→ 문제해결형 키워드 후보)
- "이불 걷어차서..."
- "뒤집기 시작하고 나서..."
- "태열이 있어서..."
- "에어컨 켜놓고 자는데..."

**구매 이유 언어** (→ 구매의도형 키워드 후보)
- "수면교육 하려고..."
- "속싸개 졸업하고..."
- "출산 선물로..."

**비교 언어** (→ 비교탐색형 키워드 후보)
- "다른 브랜드랑 비교해봤는데..."
- "메쉬 vs 밤부 고민하다가..."
