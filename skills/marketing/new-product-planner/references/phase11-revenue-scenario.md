# PHASE 11 — 수익 시나리오 모델링

## 목적

런칭부터 시장 안착까지의 **월별 매출/비용/이익 시나리오**를 3가지(보수적/기본/낙관적)로 시뮬레이션한다. 사용자가 "이 제품을 만들면 언제부터 돈이 되는지" 한눈에 볼 수 있는 재무 전망을 제공한다.

PHASE 7(가격 전략) + PHASE 10(마케팅 예산)의 데이터를 결합하여 산출.

## 사용자에게 확인할 정보

이미 수집된 정보를 정리하고, 부족한 것만 추가 확인:

| 항목 | 출처 | 없으면 |
|------|------|--------|
| 제품 원가 | PHASE 7에서 수집 | 경쟁사 가격의 30~40%로 추정 |
| 판매가 | PHASE 7에서 확정 | — |
| 마케팅 예산 | PHASE 10에서 확정 | — |
| 초기 투자비 | 사용자 확인 필요 | 금형/디자인/촬영/초도 생산 등 |
| 초도 생산 수량 | 사용자 확인 필요 | 보통 500~2,000개 |
| 재주문 리드타임 | 사용자 확인 필요 | 보통 2~4주 |
| 플랫폼 수수료 | PHASE 7에서 수집 | 네이버 10~13%, 쿠팡 10~15% |

추가 질문:
- "초기 생산 수량은 어느 정도 계획하고 계세요?"
- "금형비나 초기 셋업 비용이 있나요?"
- "재고가 다 팔리면 다음 생산까지 몇 주 걸리나요?"

## 수익 모델 구조

### 1. 매출 예측 기본 공식

```
월 매출 = 일 평균 판매량 × 판매가 × 영업일수(30일)
```

일 평균 판매량 추정 근거:
- PHASE 2.5 쇼핑 데이터 → 경쟁사 판매량 간접 추정
- PHASE 2.6 카페/블로그 언급 빈도 → 수요 수준 추정
- PHASE 5 키워드 검색량 → 구매 전환율 추정
- 네이버 쇼핑 카테고리 평균 전환율: 2~5%

### 2. 비용 구조

```
월 비용 = (원가 × 판매량) + 물류비 + 플랫폼 수수료 + 마케팅비 + 운영비
```

| 비용 항목 | 유형 | 산출 방식 |
|---------|------|---------|
| 제품 원가 | 변동비 | 원가 × 판매 수량 |
| 물류/배송비 | 변동비 | 건당 물류비 × 판매 수량 |
| 플랫폼 수수료 | 변동비 | 매출 × 수수료율 |
| 마케팅비 | (반)고정비 | PHASE 10 예산 |
| 인건비 | 고정비 | 사용자 입력 or 제외 |
| 초기 투자비 | 일회성 | 금형+디자인+촬영+초도 생산 |

### 3. 이익 계산

```
월 영업이익 = 월 매출 - 월 비용
누적 이익 = Σ(월별 영업이익) - 초기 투자비
```

## 3가지 시나리오

### 시나리오 A: 보수적 🟡
"시장이 기대보다 반응이 느린 경우"

가정:
- 런칭 첫 달 일 판매 3~5개 (월 100~150개)
- 매월 20~30% 성장
- 리뷰 확보 속도 느림 → 전환율 하위
- 마케팅 효율 ROAS 150~200%

### 시나리오 B: 기본 🟢
"계획대로 실행되는 경우"

가정:
- 런칭 첫 달 일 판매 7~10개 (월 200~300개)
- 매월 30~50% 성장 (3개월간)
- 체험단/배포형 효과 정상 발현
- 마케팅 효율 ROAS 250~350%

### 시나리오 C: 낙관적 🔵
"바이럴이 터지는 경우"

가정:
- 런칭 첫 달 일 판매 15~20개 (월 450~600개)
- 맘카페 바이럴 또는 인플루언서 콘텐츠 대박
- 매월 50~80% 성장 (초기 2개월)
- 마케팅 효율 ROAS 400%+

## 월별 P&L 테이블 (12개월)

각 시나리오별로 아래 형태의 테이블 생성:

| 월 | 판매량 | 매출 | 원가 | 물류 | 수수료 | 마케팅 | 영업이익 | 누적이익 | 마일스톤 |
|----|--------|------|------|------|--------|--------|---------|---------|---------|
| 1월차 | 200 | 960만 | 320만 | 60만 | 115만 | 250만 | 215만 | -785만 | 런칭 |
| 2월차 | 300 | 1,440만 | 480만 | 90만 | 173만 | 200만 | 497만 | -288만 | 리뷰 100건 |
| 3월차 | 400 | 1,920만 | 640만 | 120만 | 230만 | 180만 | 750만 | 462만 | ✅ BEP 달성 |
| ... | | | | | | | | | |

(위 숫자는 예시. 실제는 사용자의 원가/판매가/예산에 따라 산출)

## 핵심 마일스톤 정의

| 마일스톤 | 정의 | 기본 시나리오 예상 시점 |
|---------|------|---------------------|
| 첫 판매 | D-Day 첫 주문 | D-Day |
| 리뷰 50건 | 포토리뷰+텍스트 합산 | D+30~45 |
| 리뷰 100건 | 검색 노출 효과 시작 | D+60~75 |
| 일 판매 10개 | 일평균 두 자릿수 | D+30~60 |
| 월 매출 1,000만 | 소규모 안착 기준 | 2~3개월차 |
| BEP 달성 | 누적 이익 > 초기 투자 | 2~4개월차 |
| 월 매출 3,000만 | 중규모 안착 | 4~6개월차 |
| 시즌 재주문 | 초도 물량 소진 → 재생산 | 1~3개월차 |
| 다음 시즌 기획 | 겨울 제품 또는 내년 여름 개선 | 시즌 말 |

## 민감도 분석

핵심 변수 변화에 따른 이익 변동:

| 변수 | 변화 | BEP에 미치는 영향 |
|------|------|----------------|
| 판매가 +10% | | BEP ~N개월 앞당겨짐 |
| 판매가 -10% | | BEP ~N개월 늦어짐 |
| 마케팅비 +50% | | 초기 BEP 늦어지나 판매량 증가 기대 |
| 마케팅비 -50% | | BEP 빨라지나 성장 둔화 리스크 |
| 원가 +20% | | 마진 압박, BEP 늦어짐 |
| 전환율 2배 | | BEP 대폭 앞당겨짐 |

민감도 분석은 "어디에 돈을 더 넣으면 효과적인가"의 근거가 된다.

## 시즌별 매출 패턴 반영

PHASE 2.5 데이터랩 시즌 트렌드를 매출 예측에 반영:
- 4~5월: 런칭기 (기본 × 0.7~0.8)
- 6~7월: 성수기 피크 (기본 × 1.3~1.8)
- 8월: 성수기 후반 (기본 × 1.0~1.2)
- 9월~: 시즌 오프 (기본 × 0.3~0.5, 시즌오프 할인 매출 포함)

## 재고 관리 연계

판매 시나리오에 따른 재고 계획:
- 초도 생산: BEP 수량의 1.5~2배 권장
- 재주문 포인트: 재고 40% 남았을 때 (리드타임 고려)
- 시즌 말 재고: 전체 수량의 10% 이하 목표

## 사용자와 대화 포인트

수익 시나리오를 공유할 때:
- "보수적으로 봐도 3개월이면 투자금 회수가 가능해 보여요"
- "가장 중요한 변수는 첫 달 리뷰 확보 속도인데, 체험단 투자를 좀 더 하면 기본 시나리오 달성이 빨라질 거예요"
- "시즌 피크인 6~7월에 매출이 크게 뛰니까, 그 전에 리뷰 100건을 채우는 게 핵심이에요"
- "이 시나리오가 현실적으로 느껴지세요? 조정하고 싶은 부분 있으면 말씀해주세요"

## 리포트 반영

HTML 리포트의 "수익 시나리오" 탭:

### 구성 요소
1. **3가지 시나리오 서머리 카드**: 보수적/기본/낙관적 — 핵심 수치 비교
2. **12개월 P&L 테이블**: 시나리오 선택 탭 + 월별 상세 (접기/펼치기)
3. **매출/이익 추이 차트**: 3개 라인(시나리오별) + BEP 포인트 표시
4. **마일스톤 타임라인**: 가로 타임라인에 주요 이정표 표시
5. **민감도 분석**: 핵심 변수 슬라이더 (판매가 ±20%, 마케팅비 ±50%) → 이익 변동 실시간 표시
6. **재고 관리 알림**: 시나리오별 재주문 시점 표시
7. **투자 회수 요약**: "기본 시나리오 기준 X개월차에 BEP 달성, 첫 시즌 종료 시 누적 이익 Y원"
