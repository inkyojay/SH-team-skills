# PHASE 5 — 네이버 키워드 검색량 조사 상세 가이드

## 네이버 검색광고 API 개요

네이버 검색광고 API (keywordstool)를 사용하여 키워드의 실제 월간 검색량 데이터를 수집한다.

### API 인증 정보 (사용자에게 요청)

사용자에게 아래 3가지 정보를 요청한다:
- `NAVER_API_KEY`: 네이버 검색광고 API 라이선스 키
- `NAVER_SECRET_KEY`: 네이버 검색광고 API 시크릿 키
- `NAVER_CUSTOMER_ID`: 네이버 검색광고 고객 ID

요청 멘트: "네이버 키워드 검색량을 조회하려면 검색광고 API 인증 정보가 필요해요. API 키, 시크릿 키, 고객 ID를 알려주시겠어요? (검색광고 시스템 > API 사용 관리에서 확인 가능합니다)"

### API 엔드포인트

- URL: `https://api.searchad.naver.com/keywordstool`
- Method: GET
- Headers:
  - `X-API-KEY`: API 키
  - `X-CUSTOMER`: 고객 ID
  - `X-Signature`: HMAC-SHA256 서명 (시크릿 키 기반)

### 요청 파라미터

| 파라미터 | 설명 | 필수 |
|---------|------|------|
| hintKeywords | 쉼표 구분 키워드 (최대 5개) | Yes |
| showDetail | 1로 설정 시 상세 데이터 반환 | No |

### 응답 데이터

각 키워드별:
- `relKeyword`: 연관 키워드
- `monthlyPcQcCnt`: 월간 PC 검색량
- `monthlyMobileQcCnt`: 월간 모바일 검색량
- `monthlyAvePcClkCnt`: 월간 평균 PC 클릭수
- `monthlyAveMobileClkCnt`: 월간 평균 모바일 클릭수
- `monthlyAvePcCtr`: 월간 평균 PC 클릭률
- `monthlyAveMobileCtr`: 월간 평균 모바일 클릭률
- `plAvgDepth`: 월평균 노출 광고수
- `compIdx`: 경쟁 정도 (높음/중간/낮음)

## 실행 절차

1. PHASE 4에서 발굴한 키워드 리스트를 5개씩 묶는다 (API 제한)
2. `scripts/naver_keyword_search.py` 스크립트를 실행
3. 결과를 통합하여 정리

## 결과 정리

### 키워드 데이터 테이블

| 키워드 | 월검색량(PC) | 월검색량(모바일) | 총검색량 | 경쟁도 | 광고수 | 등급 |
|--------|------------|----------------|---------|-------|--------|------|
| | | | | | | |

### 등급 분류 기준

- **골든**: 총검색량 1,000~10,000 + 경쟁 낮음 → 최우선 공략
- **성장**: 총검색량 100~1,000 + 경쟁 낮음~중간 → 선점 기회
- **빅**: 총검색량 10,000+ → 직접 공략 어려움, 콘텐츠 참고
- **니치**: 총검색량 100 미만 → 보조 키워드, 롱테일 조합 활용

### 추가 인사이트

- **모바일 비중**: 모바일 검색량 / 총 검색량 → 모바일 최적화 필요 여부
- **클릭률**: 검색 대비 실제 클릭 비율 → 구매 의도 간접 지표
- **광고 경쟁**: 노출 광고수 많으면 유료 광고 진입 어려움 → 오가닉 SEO 집중

## API 실패 시 대안

네이버 API가 작동하지 않을 경우:
1. `web_search`로 "네이버 키워드 도구 [키워드]" 검색
2. 블랙키위, 키워드마스터 등 서드파티 도구 참조
3. 네이버 자동완성 결과로 상대적 인기도 추정
4. 사용자에게 수동 조회를 안내하고 결과를 입력받는 방식으로 대체
