# PHASE 3 — 레퍼런스 이미지 수집 가이드

## 이미지 수집 목적

신제품 기획 보고서에 시각적 근거를 제공하여:
- 경쟁사 제품이 실제로 어떻게 생겼는지 한눈에 파악
- 디자인/패키징/상세페이지의 방향성 레퍼런스 확보
- 경영진/팀원에게 "이런 느낌의 제품을 만들려 한다"를 직관적으로 전달

## 방식 1: 자동 이미지 검색

`image_search` 도구를 활용하여 아래 카테고리별로 이미지를 수집한다.

### 검색 쿼리 패턴

**경쟁사 제품 이미지:**
- "[경쟁사 브랜드] [제품명] 제품 사진"
- "[경쟁사 브랜드] [카테고리] 상세페이지"

**카테고리 레퍼런스:**
- "[카테고리] 제품 디자인 레퍼런스"
- "[카테고리] 패키징 디자인"
- "baby cooling mat product design"  (해외 레퍼런스)

**디자인/무드보드:**
- "[카테고리] 미니멀 디자인"
- "[카테고리] 프리미엄 패키징"

### 수집 시 주의사항
- 경쟁사 제품 사진은 공개된 쇼핑몰/상세페이지 이미지만 활용
- 해외 레퍼런스는 디자인 방향성 참고용으로만 수집
- 저작권 주의: 리포트에 포함 시 "레퍼런스 참고용" 명시
- 이미지당 출처 URL을 함께 기록

### 수집 수량 가이드
- 경쟁사별 대표 제품 이미지: 1~2장씩 (총 5~10장)
- 해외 레퍼런스: 3~5장
- 패키징/상세페이지 레퍼런스: 3~5장
- 전체 15~20장 내외가 적정

## 방식 2: 사용자 직접 업로드

사용자가 직접 수집한 레퍼런스 이미지를 업로드하여 리포트에 통합한다.

### 안내 멘트
"레퍼런스로 사용하고 싶은 제품 사진이나 디자인 이미지가 있으시면 업로드해주세요.
경쟁사 제품 사진, 해외 레퍼런스, 원하는 디자인 무드 이미지 등 뭐든 괜찮아요."

### 업로드 이미지 처리
1. 업로드된 이미지는 `/mnt/user-data/uploads/`에 저장됨
2. `view` 도구로 이미지 확인
3. 사용자에게 각 이미지의 카테고리/설명을 물어봄:
   - "이 이미지는 어떤 용도인가요? (경쟁사 제품 / 디자인 레퍼런스 / 소재 참고 / 기타)"
4. 리포트의 레퍼런스 보드 탭에 카테고리별로 정리하여 삽입

### 이미지 삽입 형식 (HTML 리포트)
```html
<div class="ref-grid">
  <div class="ref-item">
    <img src="[이미지 URL 또는 base64]" alt="[설명]">
    <div class="ref-label">[카테고리] — [설명]</div>
    <a href="[출처 URL]">출처 보기</a>
  </div>
</div>
```

## 이미지 수집 타이밍

- PHASE 3 (경쟁사 분석) 중에 경쟁사 제품 이미지 자동 수집
- PHASE 3 완료 후 사용자에게 추가 레퍼런스 업로드 요청
- PHASE 6 (리포트 생성) 시 모든 이미지를 통합 배치

## 두 방식의 조합

이상적인 흐름:
1. Claude가 자동으로 경쟁사 제품 + 해외 레퍼런스 이미지를 검색하여 먼저 수집
2. 수집 결과를 사용자에게 보여주며 "이 외에 추가하고 싶은 이미지 있으세요?"
3. 사용자가 업로드하면 함께 통합
4. 최종 리포트의 "레퍼런스 보드" 탭에 모두 정리
