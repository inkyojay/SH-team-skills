# 신제품 기획 플래너

신제품을 기획할 때 아이디어 발상부터 시장 검증까지 전체 프로세스를 대화형으로 진행하는 스킬.
**핵심 철학**: 사용자와 함께 아이디어를 발전시키며, 데이터(검색량/경쟁사)로 검증하고, 실행 가능한 제품 컨셉으로 구체화한다.

## 브랜드 컨텍스트 (기본값 — 썬데이허그)

- **브랜드**: 썬데이허그 (Sunday Hug) / 주식회사 제이코프 (JAYCORP)
- **카테고리**: 아기 수면용품 (슬리핑백, 속싸개 등) + 확장 가능
- **포지셔닝**: "입는 이불" — 국내산 프리미엄 아기 수면 케어
- **타겟**: 0~5세 자녀를 둔 30대 부모 (특히 엄마)
- **강점**: 국내 자체 생산, 소재 투명성, 단계별 성장 설계

범용 모드에서는 사용자에게 브랜드/카테고리/타겟 정보를 먼저 수집한다.

---

## 실행 프로세스 — 13단계 (기획 → 런칭 → 수익 시뮬레이션)

이 스킬은 **대화형**으로 진행한다. 각 단계마다 사용자와 소통하며 방향을 잡는다.
단, 사용자가 "한 번에 다 해줘"라고 하면 최소한의 확인만 거치고 빠르게 진행할 수 있다.

### PHASE 0 — 시장 기회 탐색 (Market Opportunity Discovery)

**목적**: 사용자에게 구체적 아이디어가 없을 때, 데이터 기반으로 유망한 제품 기회를 발굴한다.

이 단계는 **선택적**이다. 사용자가 이미 아이디어가 있으면 PHASE 1로 바로 진행.
사용자가 "뭘 만들지 모르겠다", "기회 찾아줘", "해외에서 핫한 거" 등을 말하면 이 단계를 실행한다.

**5가지 기회 탐색 소스**:

1. **해외 트렌드 역수입 기회**: 미국/유럽/일본 Amazon, 해외 육아 커뮤니티에서 인기인데 국내에 없는 제품. web_search로 "best baby products 2025/2026 USA/UK/Japan", "trending baby items Amazon", "baby product innovation" 등 조사.

2. **국내 VOC 미충족 수요**: 맘카페/지식인에서 반복되는 불만 중 아직 해결된 제품이 없는 것. "이런 제품 없나요?", "해외에서 사야 해요" 패턴 탐색.

3. **카테고리 갭 분석**: 썬데이허그의 기존 라인업(슬리핑백, 속싸개, 수면용품)에서 빠진 연관 카테고리. 고객 여정(임신→출산→신생아→영아→유아)에서 수면 외 확장 가능 영역.

4. **시즌/시기 기회**: 현재 시점에서 3~6개월 후 시즌에 맞는 제품. 기존 비수기를 채울 수 있는 제품.

5. **크로스보더 베스트셀러**: Amazon Baby Best Sellers, 일본 라쿠텐 베이비 랭킹에서 국내 미출시 카테고리 스캔.

**실행 방법**:
- 3~5개 에이전트를 병렬로 실행하여 각 소스에서 기회를 수집
- 수집된 기회를 **기회 스코어카드**로 평가:

| 평가 항목 | 가중치 |
|----------|--------|
| 해외 검증도 (실제 판매 실적) | 25% |
| 국내 시장 공백 (경쟁 부재) | 25% |
| 썬데이허그 브랜드 적합성 | 20% |
| 제조 가능성 (기존 생산 역량 활용) | 15% |
| 시장 규모 잠재력 | 15% |

- 상위 5~7개 기회를 사용자에게 제시하고 선택하게 한다
- 선택된 기회로 PHASE 1 브레인스토밍 진행

### PHASE 1 — 아이디어 브레인스토밍
→ 세부 가이드: `references/phase1-brainstorming.md`

**목적**: 사용자의 머릿속 아이디어를 꺼내고, 함께 확장하고, 유망한 방향 3~5개로 좁히기

진행 방식:
1. **시작 질문**: "어떤 제품을 생각하고 계신가요?" — 구체적이든 막연하든 OK
2. **확장 대화**: 사용자의 답변을 바탕으로 관련 트렌드, 시장 기회, 타겟 니즈를 제시하며 아이디어를 확장
3. **구조화**: 나온 아이디어들을 아래 프레임워크로 정리

아이디어 정리 프레임워크:
| 항목 | 내용 |
|------|------|
| 제품 컨셉 | 한 줄 설명 |
| 타겟 고객 | 누구를 위한 제품인가 |
| 해결하는 문제 | 고객의 어떤 페인포인트를 해결하나 |
| 차별화 포인트 | 기존 제품과 뭐가 다른가 |
| 시즌/시기 | 언제 가장 잘 팔릴 것인가 |

4. **선택**: 사용자에게 가장 매력적인 아이디어 1~2개를 선택하게 한다

---

### PHASE 2 — 시장 조사 & 트렌드 분석
→ 세부 가이드: `references/phase2-market-research.md`

**목적**: 선택한 아이디어의 시장성을 데이터로 확인

조사 항목:
1. **시장 규모**: 해당 카테고리 국내/글로벌 시장 규모, 성장률
2. **트렌드**: 최근 6개월~1년 사이 떠오르는 키워드/트렌드
3. **소비자 니즈**: 맘카페, 커뮤니티, 리뷰에서 반복되는 불만/요구
4. **계절성**: 시즌별 수요 변화 패턴
5. **규제/인증**: 필요한 인증, 안전 기준 확인

실행 방법:
- `web_search`로 시장 리포트, 뉴스, 트렌드 기사 수집
- `web_fetch`로 주요 기사/리포트 상세 내용 확인
- 결과를 사용자에게 공유하며 "이 시장 괜찮은 것 같은데, 어떻게 보세요?" 식으로 대화

---

### PHASE 2.5 — 네이버 쇼핑 데이터 분석 (시장 경쟁력 검증)
→ 세부 가이드: `references/phase2-5-shopping-analysis.md`
→ 실행 스크립트: `scripts/naver_shopping_analysis.py`

**목적**: 네이버 쇼핑 API 데이터로 시장 경쟁력과 제품 성공 가능성을 정량적으로 검증

네이버 개발자센터(developers.naver.com) Client ID / Secret이 필요 (검색광고 API와 별도).

5가지 분석:
1. **경쟁사 제품 자동 수집**: 쇼핑 검색 → 상위 100개 상품명/가격/브랜드/이미지/링크 한꺼번에 수집
2. **가격대 분석**: 최저~최고가 분포 → 가성비/주류/프리미엄 3구간 가격 포지셔닝 추천
3. **시즌별 수요 트렌드**: 데이터랩 월별 검색 클릭 추이 → 성수기/비수기 + 출시 타이밍
4. **타겟 고객 검증**: 성별/연령별 검색 비율 → "30대 여성이 실제 주 검색층" 데이터 확인
5. **브랜드 점유율 추정**: 검색 결과 내 브랜드 등장 빈도 → 시장 집중도/진입 기회 판단

이 분석 결과는 PHASE 3 경쟁사 분석, PHASE 6 사업 타당성 섹션에 데이터 근거로 직접 반영.

Claude 환경에서 API 차단 시: web_search 기반 대체 + 로컬 PC 실행 안내.

### PHASE 2.6 — 고객 목소리(VOC) 통합 분석
→ 세부 가이드: `references/phase2-6-voc-analysis.md`
→ 실행 스크립트: `scripts/naver_voc_analysis.py`

**목적**: 맘카페, 블로그, 지식인, 뉴스에서 실제 고객의 목소리를 데이터로 수집하여 시장을 소비자 관점에서 검증

동일한 네이버 개발자센터 Client ID/Secret으로 5가지 분석:
1. **카페글(맘카페) 분석**: 실제 부모의 고민/추천/불만 패턴 수집 → 제품 개선 기회 발견
2. **블로그 리뷰 분석**: 체험단 vs 진성 리뷰 비율, 경쟁사 리뷰 키워드 → 마케팅 벤치마크
3. **지식인 질문 분석**: 구매 전 질문 유형(추천/비교/안전/가격) → 상세페이지 FAQ 소재 + 롱테일 키워드 후보
4. **뉴스 동향 수집**: 업계 뉴스, 신소재, 경쟁사 보도 → 시장 동향 근거
5. **검색어 트렌드**: 키워드 상대 검색량 추이(0~100) → 성수기 파악 + PHASE 2.5와 교차 검증

VOC 데이터는 리포트의 시장 분석, 경쟁사 분석, 사업 타당성 섹션에 "고객이 실제로 이렇게 말하고 있다"는 근거로 반영.

### PHASE 3 — 경쟁사 분석 + 레퍼런스 수집
→ 세부 가이드: `references/phase3-competitor-analysis.md`

**목적**: 경쟁 제품의 강점/약점을 파악하고, 우리의 차별화 포인트 확보. 실제 보고서에 활용할 수 있도록 경쟁사 제품 링크와 레퍼런스 이미지도 함께 수집한다.

분석 프레임:
1. **직접 경쟁사 3~5개**: 동일 카테고리 핵심 플레이어
2. **간접 경쟁사 2~3개**: 대체재를 제공하는 플레이어
3. **각 경쟁사별 분석 항목**:
   - 제품 라인업 & 가격대
   - USP / 핵심 소구점
   - 고객 리뷰에서 나오는 장점/단점
   - 마케팅 채널 & 전략
   - 약점 = 우리의 기회
   - **제품 상세페이지 URL** (리포트에 직접 링크 삽입)

실행 방법:
- `web_search`로 경쟁사 제품, 리뷰, 마케팅 현황 조사
- `web_fetch`로 경쟁사 상세페이지, 리뷰 페이지 크롤링
- **경쟁사 제품 URL을 반드시 수집**하여 리포트에 클릭 가능한 링크로 포함
- 경쟁사 비교표를 만들어 사용자와 공유
- **핵심 질문**: "경쟁사 대비 우리가 이길 수 있는 포인트가 뭘까요?"

**레퍼런스 이미지 수집 (2가지 방식)**:
→ 세부 가이드: `references/phase3-reference-images.md`

1. **자동 검색**: `image_search` 도구로 경쟁사 제품, 유사 카테고리 제품, 해외 레퍼런스 이미지를 검색하여 리포트에 포함
   - 경쟁사별 대표 제품 이미지
   - 해외 유사 제품 디자인 레퍼런스
   - 패키징/상세페이지 레퍼런스
2. **사용자 업로드**: 사용자가 직접 모은 레퍼런스 이미지를 업로드하면 리포트에 통합
   - 업로드된 이미지는 `/mnt/user-data/uploads/`에서 읽어 리포트에 삽입
   - "레퍼런스 이미지 있으시면 업로드해주세요"라고 안내

각 단계에서 이미지를 수집/업로드 받으면 메모리에 보관하다가 PHASE 6 리포트에 통합한다.

**신제품 무드보드 생성**:
→ 세부 가이드: `references/phase3-moodboard.md`
→ 생성 스크립트: `scripts/generate_moodboard.py` (brand-dna-extractor 연동)

PHASE 3 진행 중 수집된 이미지 + 컬러 팔레트 + 무드 키워드를 조합하여 **시각적 무드보드**를 자동 생성한다.

무드보드 소재:
- **이미지**: 경쟁사 제품(PHASE 2.5 쇼핑 API image 필드) + 레퍼런스(image_search) + 사용자 업로드
- **컬러 팔레트**: 브랜드 컬러 + 경쟁사 트렌드 컬러 + 시즌 컬러
- **무드 키워드**: PHASE 8 소구점 + PHASE 2.6 VOC 감성 단어 + 사용자 지정

생성 타이밍:

키워드 발굴 4가지 소스:

| 정보탐색형 | 일반 정보 | 중간 | "아기 적정 수면 온도" |

산출물: 키워드 리스트 (키워드 / 검색의도 / 예상 경쟁도 / 활용 채널)

### PHASE 5 — 네이버 키워드 검색량 조사
→ 세부 가이드: `references/phase5-naver-keyword-api.md`
→ 실행 스크립트: `scripts/naver_keyword_search.py`

**목적**: PHASE 4에서 발굴한 키워드의 실제 네이버 검색량을 데이터로 확인

실행 절차:
1. 사용자에게 네이버 검색광고 API 인증 정보 요청:
   - `NAVER_API_KEY` (API 키)
   - `NAVER_SECRET_KEY` (시크릿 키)  
   - `NAVER_CUSTOMER_ID` (고객 ID)
2. `scripts/naver_keyword_search.py` 스크립트 실행
3. 반환 데이터: 월간 검색량(PC/모바일), 월평균 클릭수, 경쟁 정도, 월평균 노출 광고수

API 호출 시 키워드를 5개씩 묶어서 배치 처리 (API 제한 고려).

**네이버 API 환경별 실행 방법**:
→ 세부 가이드: `references/phase5-local-execution.md`

- **Claude 환경**: `api.searchad.naver.com`이 네트워크 정책에 의해 차단될 수 있음. 이 경우 web_search 기반으로 대체 검색량 데이터를 추정하고, 리포트에 "로컬 실행 가이드"를 포함하여 사용자가 직접 정확한 데이터를 확보할 수 있게 안내한다.
- **로컬 PC**: Python 3.8+ 환경에서 스크립트를 직접 실행하면 정확한 검색량 데이터를 얻을 수 있음. 리포트에 실행 명령어와 결과를 리포트에 붙여넣는 방법을 안내한다.

검색량 기반 우선순위:
- **골든 키워드**: 검색량 1,000~10,000 + 경쟁 낮음 → 1순위 공략
- **성장 키워드**: 검색량 100~1,000 + 상승 추세 → 선점 기회
- **빅 키워드**: 검색량 10,000+ → 콘텐츠 참고용 (직접 공략 어려움)
- **니치 키워드**: 검색량 100 미만 → 보조 키워드로 활용

사용자에게 결과 공유 시: "이 키워드들 중에서 가장 기회가 큰 것 같은 건 OOO인데, 어떻게 생각하세요?"

### PHASE 7 — 가격 전략 수립
→ 세부 가이드: `references/phase7-pricing-strategy.md`

5. **손익분기점(BEP)**: 시나리오별 BEP 수량 + 달성 예상 시기

### PHASE 8 — 제품 마케팅 소구점 설계
→ 세부 가이드: `references/phase8-appeal-points.md`

**목적**: PHASE 1~7의 모든 데이터를 마케팅에서 바로 사용할 소구점으로 변환

4계층 소구점:
1. **기능적 (Functional)**: 냉감 효과, 보풀 방지, 논슬립 → 경쟁사 약점 해결 근거 (PHASE 2.6 VOC)
2. **감성적 (Emotional)**: 아기 꿀잠, 엄마 안심 → 맘카페 만족 키워드에서 추출
3. **사회적 (Social Proof)**: 리뷰 수, 맘카페 인기, 브랜드 실적 → 신뢰 구축
4. **긴급성 (Urgency)**: 시즌 한정, 얼리버드, 세트 한정 → 구매 촉진

각 소구점마다: 근거 데이터 + 예시 카피 + 활용 채널 매핑 포함.
상세페이지 스토리라인(훅→문제→해결→신뢰→CTA)도 함께 설계.

### PHASE 10 — 마케팅 예산 상세 설계 & 인플루언서 전략
→ 세부 가이드: `references/phase10-marketing-budget.md`

**목적**: 런칭 전략을 실제 집행 가능한 예산 계획으로 전환 + 인플루언서 마케팅 구체 설계

이 단계는 **사용자와 대화형**으로 진행. 예산은 회사마다 다르므로 반복 조율한다.

사용자에게 확인:
- 총 마케팅 예산 (또는 매출 대비 비율)
- 인플루언서 마케팅 경험/선호 (체험단 위주? 유료 인플루언서도?)
- 내부 리소스 현황 (콘텐츠 자체 제작? 외주?)
- 광고 집행 의향 (네이버 쇼핑 광고, 인스타 광고)

산출물:
1. **마케팅 비용 항목별 산출**: 콘텐츠 제작비 + 인플루언서/체험단 + 광고비 + 프로모션 비용
2. **인플루언서 티어별 전략**: 나노/마이크로/미디엄/매크로 — 선정 기준, 협업 유형, 단가, 일정 배치
3. **예산 시나리오 3종**: 소규모(100만↓) / 중규모(100~300만) / 대규모(300만↑) 배분 안
4. **인플루언서 협업 유형**: 시딩/유료포스팅/체험단/앰배서더/라이브커머스/공동기획
5. **월별 예산 집행 캘린더**: PHASE 9 액션플랜과 연동

### PHASE 11 — 수익 시나리오 모델링
→ 세부 가이드: `references/phase11-revenue-scenario.md`

**목적**: 런칭~시장 안착까지 월별 매출/비용/이익을 3가지 시나리오로 시뮬레이션

PHASE 7(가격) + PHASE 10(마케팅 예산) 데이터를 결합하여 산출.

사용자에게 추가 확인: 초기 투자비(금형/디자인/촬영), 초도 생산 수량, 재주문 리드타임

3가지 시나리오:
- **보수적 🟡**: 시장 반응 느림, 일 판매 3~5개 시작, ROAS 150~200%
- **기본 🟢**: 계획대로 실행, 일 판매 7~10개 시작, ROAS 250~350%  
- **낙관적 🔵**: 바이럴 터짐, 일 판매 15~20개 시작, ROAS 400%+

산출물:
1. **12개월 P&L 테이블**: 시나리오별 월간 매출/원가/물류/수수료/마케팅/영업이익/누적이익
2. **BEP(손익분기점) 분석**: 시나리오별 BEP 수량 + 달성 예상 시점
3. **마일스톤 타임라인**: 리뷰 50건/100건, 일 판매 10개, 월 매출 1000만/3000만, BEP 달성
4. **민감도 분석**: 판매가/마케팅비/전환율 변화에 따른 이익 변동
5. **시즌 패턴 반영**: PHASE 2.5 트렌드 데이터로 성수기/비수기 매출 보정
6. **재고 관리 알림**: 시나리오별 재주문 시점 + 시즌 말 잔여 재고 추정

### 최종 (선택): PPT형 PDF 보고서 변환

HTML 리포트 생성 후, 사용자에게 다음과 같이 물어본다:
"HTML 리포트가 완성됐어요! **PPT 스타일 PDF 보고서**로도 변환할까요? 팀 보고/공유용으로 좋습니다."

사용자가 원하면 `/pptx` 스킬을 호출하여 **프레젠테이션형 PDF**를 생성한다.

**PPT 구성 (약 25~35슬라이드)**:

| 슬라이드 | 내용 |
|----------|------|
| 1 | **표지** — 기획명, 브랜드 로고, 날짜, 작성자 |
| 2 | **목차** — 전체 섹션 한눈에 |
| 3~4 | **Executive Summary** — 핵심 수치 4개 + 3줄 요약 + Go/No-Go 결론 |
| 5~6 | **제품 컨셉** — 제품 비교표 + 디자인 방향 + 무드보드 |
| 7~8 | **사업 타당성** — 시장 기회 + 브랜드 적합성 + 리스크 대응 |
| 9~10 | **시장 분석** — 시장 규모 차트 + VOC 핵심 인사이트 + 트렌드 |
| 11~12 | **경쟁사 분석** — 비교표 + 포지셔닝 맵 + 우리 기회 |
| 13 | **키워드 분석** — TOP 10 키워드 + 시즌 트렌드 차트 |
| 14~15 | **가격 전략** — 원가 워터폴 + 가격 포지셔닝 + 세트 가격 + BEP |
| 16~17 | **마케팅 소구점** — 4계층 소구점 + 상세페이지 플로우 |
| 18~19 | **런칭 전략** — 3단계 타임라인 + 채널별 역할 |
| 20~21 | **마케팅 예산** — 비용 산출 + 인플루언서 전략 + 예산 시나리오 |
| 22~23 | **수익 시나리오** — 3종 P&L 요약 + BEP + 민감도 |
| 24 | **주요 마일스톤** — 12주 타임라인 시각화 |
| 25 | **Next Steps** — 즉시 실행 액션 아이템 3~5개 |

**디자인 가이드**:
- 한 슬라이드에 내용 과적 금지 — 핵심 수치 + 시각화 위주
- 슬라이드당 텍스트 5줄 이내, 나머지는 표/차트로
- 컬러: 해당 기획의 HTML 리포트와 동일 색상 체계 사용
- 폰트: 깔끔한 고딕 계열
- 모든 차트/표는 HTML 리포트의 데이터를 그대로 사용

**실행 방법 — python-pptx 직접 사용** (html2pptx가 아닌 python-pptx로 직접 생성):

python-pptx를 사용하여 PPT를 생성한다. html2pptx 변환 방식은 빈 공간이 많고 디자인이 빈약하므로 사용하지 않는다.

```python
from pptx import Presentation
from pptx.util import Inches, Pt, Emu
from pptx.dml.color import RGBColor
from pptx.enum.text import PP_ALIGN, MSO_ANCHOR
from pptx.enum.chart import XL_CHART_TYPE
```

핵심 디자인 규칙:
1. **배경 꽉 채우기**: 모든 슬라이드에 배경색 또는 컬러 블록을 적용하여 빈 공간 최소화
2. **좌측 컬러 바**: 각 슬라이드 왼쪽에 포인트 컬러 바(width 0.4인치) 배치 → 시각적 통일감
3. **헤더 영역**: 상단 1.2인치를 진한 배경 + 흰색 텍스트로 제목 영역 구성
4. **카드 레이아웃**: 핵심 수치는 컬러 배경 도형(rounded rect) 안에 큰 숫자 + 작은 설명
5. **표**: python-pptx Table 객체로 생성, 헤더 행은 진한 배경 + 흰색 텍스트, 데이터 행은 교차 줄무늬
6. **차트**: python-pptx Chart 객체로 생성 (수정 가능한 엑셀 연동 차트)
7. **여백**: 좌우 0.5인치, 상하 적절히 → 빈 공간 없이 콘텐츠 배치
8. **폰트**: 제목 24~28pt 볼드, 본문 12~14pt, 수치 강조 36~48pt
9. **컬러**: 메인 2색 + 포인트 1색 + 흰색/회색 조합
10. **한 슬라이드 = 한 메시지**: 제목(What) + 핵심 수치/표/차트(Evidence) + 인사이트 한줄(So What)

구현 패턴:
```python
# 슬라이드 기본 구조
slide = prs.slides.add_slide(prs.slide_layouts[6])  # 빈 레이아웃 사용

# 상단 헤더 바
header = slide.shapes.add_shape(MSO_SHAPE.RECTANGLE, 0, 0, prs.slide_width, Inches(1.2))
header.fill.solid()
header.fill.fore_color.rgb = RGBColor(0x1B, 0x2A, 0x4A)  # 메인 컬러
header.line.fill.background()

# 좌측 포인트 바
left_bar = slide.shapes.add_shape(MSO_SHAPE.RECTANGLE, 0, 0, Inches(0.3), prs.slide_height)
left_bar.fill.solid()
left_bar.fill.fore_color.rgb = RGBColor(0x2E, 0xC4, 0xB6)  # 포인트 컬러

# 제목 텍스트
title_box = slide.shapes.add_textbox(Inches(0.5), Inches(0.2), Inches(9), Inches(0.8))
tf = title_box.text_frame
p = tf.paragraphs[0]
p.text = "슬라이드 제목"
p.font.size = Pt(28)
p.font.bold = True
p.font.color.rgb = RGBColor(0xFF, 0xFF, 0xFF)
```

저장 및 PDF 변환:
1. .pptx 저장: `prs.save(path)`
2. PDF 변환: `soffice --headless --convert-to pdf --outdir {dir} {pptx_path}`
3. 저장 위치: `{기획명}/01_리포트/{기획명}_보고서.pptx` + `{기획명}_보고서.pdf`

**트리거**: 사용자가 "PDF로", "PPT로", "보고서로", "발표자료", "팀 공유용" 등을 말하면 이 단계를 실행한다.

### 데이터 이중 저장 규칙
- **JSON**: 프로그래밍/재작업용 원본. 스크립트 실행 시 항상 JSON으로 먼저 저장.
- **CSV**: 사용자 열람용. JSON 생성 후 자동으로 CSV도 함께 생성한다.
  - CSV는 `utf-8-sig` 인코딩 (엑셀에서 한글 깨짐 방지)
  - 파일명은 한글로 내용을 알 수 있게 작성 (예: `슬리핑백_경쟁제품_100개.csv`)
  - JSON 파일명은 영문 snake_case 유지 (예: `data_sleeping_bag.json`)

### 폴더 생성 시점
- PHASE 1에서 제품 기획명이 확정되면 즉시 폴더 구조를 생성한다.
- 기획명은 사용자에게 확인받거나, 브랜드+소재+시즌+제품 수 조합으로 자동 생성한다.

## 주의사항

- **대화의 흐름을 유지한다**: 스킬이 작동하더라도 기계적으로 진행하지 않고, 자연스러운 기획 회의 분위기를 만든다
- **사용자 의견을 존중한다**: 데이터가 부정적이어도 사용자가 강하게 원하면 리스크를 알려주되 진행한다
- **중간 저장**: 각 단계 결과물은 메모리에 유지하고, 마지막에 HTML로 통합
- **네이버 API 실패 시**: web_search로 "키워드 + 네이버 검색량" 검색하여 대체 데이터 확보
- **경쟁사 정보 출처**: 항상 공개된 정보(웹사이트, 리뷰, 뉴스)만 활용
