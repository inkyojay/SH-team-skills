# 상품 키워드 최적화 스킬

기존 상품의 네이버 쇼핑 노출을 극대화하기 위해, 데이터 기반으로 최적의 키워드 조합을 발굴하는 스킬.

## 핵심 원칙

- **블루 키워드**: 검색량은 충분하고, 경쟁은 낮은 키워드를 찾는다
- **고객 언어**: 판매자 용어가 아닌, 고객이 실제 검색하는 표현을 쓴다
- **데이터 우선**: 감이 아닌 검색량/경쟁도/클릭률 수치로 판단한다

## 워크플로우

사용자에게 먼저 다음을 확인한다:
1. **상품명** (또는 상품 카테고리)
2. **현재 사용 중인 키워드** (있으면)
3. **타겟 고객** (연령, 성별 등)

---

### STEP 1. 시드 키워드 수집

상품 정보로부터 초기 키워드 후보를 만든다.

#### 1-1. 상품 속성 분해

상품을 속성별로 분해한다:
- **카테고리**: 슬리핑백, 보행기, 젖병 등
- **소재/특성**: 오가닉, 메쉬, 실리콘, 항균 등
- **타겟**: 신생아, 돌아기, 임산부 등
- **시즌/용도**: 여름용, 사계절, 외출용, 실내용 등
- **브랜드 키워드**: 썬데이허그 등

조합 공식:
```
[타겟] + [소재/특성] + [카테고리] + [시즌/용도]
→ "신생아 오가닉 슬리핑백 여름용"
```

최소 **15개** 시드 키워드를 생성한다.

#### 1-2. 네이버 쇼핑 경쟁 상품 역추적

MCP 도구 `mcp__naver-search__search_shop`으로 메인 키워드 검색 후, 상위 상품들의:
- 상품명에 포함된 키워드 추출
- 카테고리 태그 확인
- 반복 등장하는 표현 수집

```
예: "슬리핑백" 검색 → 상위 10개 상품명에서 공통 키워드 추출
→ "사계절", "겉싸개", "신생아", "퇴원", "속싸개" 등 발견
```

#### 1-3. 고객 언어 수집

실제 고객이 사용하는 표현을 수집한다:

- `mcp__naver-search__search_kin`: 지식인에서 "[카테고리] 추천" 검색 → 질문 제목의 표현 수집
- `mcp__naver-search__search_cafearticle`: 맘카페에서 "[카테고리] 어떤거" 검색 → 실사용 표현 수집
- `mcp__naver-search__search_blog`: 블로그에서 "[카테고리] 비교" 검색 → 리뷰어 표현 수집

수집 포인트:
- 질문 제목의 자연어 표현 (예: "신생아 잘때 뭐 입혀요" → "신생아 수면 옷")
- 브랜드명 없이 검색하는 패턴
- "~추천", "~비교", "~차이", "~후기" 패턴

### STEP 2. 검색량 + 경쟁도 조회

수집한 키워드 후보를 네이버 검색광고 API로 정량 분석한다.

#### 실행 방법

```bash
python3 ~/Projects/team-skills/skills/new-product-planner/scripts/naver_keyword_search.py \
  --api-key "$NAVER_API_KEY" \
  --secret-key "$NAVER_SECRET_KEY" \
  --customer-id "$NAVER_CUSTOMER_ID" \
  --keywords "키워드1,키워드2,키워드3,키워드4,키워드5" \
  --output /tmp/keyword_result.json
```

API 키가 환경변수에 없으면 사용자에게 요청한다:
> "네이버 검색광고 API 키가 필요합니다. 검색광고 시스템 > API 사용 관리에서 API 키, 시크릿 키, 고객 ID를 확인해주세요."

#### 수집 데이터

| 항목 | 설명 | 활용 |
|------|------|------|
| monthlyPcQcCnt | 월간 PC 검색량 | 검색 볼륨 |
| monthlyMobileQcCnt | 월간 모바일 검색량 | 검색 볼륨 (주력) |
| monthlyAveMobileClkCnt | 월간 모바일 클릭수 | 실제 유입 가능성 |
| monthlyAveMobileCtr | 모바일 클릭률 | 구매 의도 판단 |
| compIdx | 경쟁 정도 | 높음/중간/낮음 |
| plAvgDepth | 평균 광고 노출수 | 광고 경쟁 강도 |
| relKeyword | 연관 키워드 | 추가 후보 발굴 |

**중요**: API가 반환하는 `relKeyword` (연관 키워드)도 추가 후보로 수집한다. 이것이 핵심 확장 소스다.

### STEP 3. 쇼핑 트렌드 분석 (DataLab)

MCP 도구로 키워드의 쇼핑 트렌드를 분석한다.

#### 3-1. 카테고리 코드 찾기

`mcp__naver-search__find_category`로 상품 카테고리 코드를 먼저 조회한다.

#### 3-2. 키워드 비교

`mcp__naver-search__datalab_shopping_keywords`로 후보 키워드 간 쇼핑 트렌드를 비교한다.

```
예: "슬리핑백" vs "아기슬리핑백" vs "신생아슬리핑백"
→ 최근 6개월 트렌드 비교
```

#### 3-3. 타겟 분석 (선택)

필요시 성별/연령/디바이스별 분석:
- `datalab_shopping_keyword_by_gender`: 성별 검색 비중
- `datalab_shopping_keyword_by_age`: 연령대별 검색 비중
- `datalab_shopping_keyword_by_device`: PC vs 모바일 비중

---

### STEP 4. 키워드 스코어링

수집한 데이터로 각 키워드에 점수를 매긴다.

#### 스코어링 공식

```
키워드 점수 = (월간 모바일 검색량 × 모바일 클릭률) / 경쟁 지수
```

경쟁 지수 변환:
- 높음 = 3
- 중간 = 2
- 낮음 = 1

#### 분류 기준

| 등급 | 조건 | 활용 |
|------|------|------|
| S (블루 키워드) | 검색량 1000+ & 경쟁 낮음 | 최우선 공략 |
| A (핵심 키워드) | 검색량 5000+ & 경쟁 중간 | 메인 키워드 |
| B (보조 키워드) | 검색량 500~5000 & 경쟁 중간 이하 | 서브 키워드 |
| C (롱테일) | 검색량 100~500 & 경쟁 낮음 | 상세설명/태그 활용 |
| D (비추천) | 검색량 100 미만 또는 경쟁 높음 & 검색량 낮음 | 제외 |

---

### STEP 5. 키워드 스코어링 결과

각 시드 + 연관 키워드를 등급(S/A/B/C/D)으로 분류한다. 등급 기준은 STEP 4 참조.

---

### STEP 6. 스마트스토어 상품명 수정안 생성 (3가지)

**네이버 공식 SEO 규칙** (반드시 준수):

| 규칙 | 내용 |
|------|------|
| 글자수 | 50자 내외 권장 (최대 100자) |
| 조사/수식어 금지 | "추천", "인기", "한정", "특가", "즉시할인", "재입고", "신상품", "주문폭주" → 감점 요인 |
| 중복 단어 피하기 | 같은 단어 2번 이상 반복 X |
| 언어 | 한글 + 필요시 영문, 숫자는 아라비아 |
| 띄어쓰기 | 키워드 단위로 띄어 쓰기 (붙여쓰기 X) |
| 홍보성 제거 | 홍보 수식어 제거 |

**3가지 상품명 안 구성**:

1. **① 밸런스형 (권장)** — A등급 4~5개 + C등급 롱테일 1개 + 사이즈
2. **② 검색량 최대화형** — 최고 검색량 A등급을 앞에 배치
3. **③ 블루오션 특화형** — B등급 블루 키워드 전진 + 소재/시즌 명시

각 안에 대해 다음을 기재한다:
- 글자수
- 포함 키워드 등급 (A/B/C)
- 타겟 상황 (시즌, 사이즈 등)
- SEO 규칙 준수 여부 체크

---

### STEP 7. 상품태그 10개 추천 (네이버 태그사전 기반)

**네이버 스마트스토어 상품태그 규칙**:
- 태그사전에 등록된 태그만 유효 (등록 안 된 태그는 무효)
- 최대 10개 반영 (초과 입력해도 10개만)
- 콤마(,) 구분으로 입력
- **유효성 확인**: 등록 후 `"상품명 + 태그"` 검색 → "상품명+태그로만 검색하기" 노출되면 유효

**태그사전 등록 가능성 판단 기준** (3가지 충족 시 등록 확률 매우 높음):
1. 경쟁사 상품명에 반복 등장 (쇼핑 검색 Top 15 중 3회+)
2. 검색량 100회/월 이상 (검색광고 API 기준)
3. 네이버 쇼핑 카테고리와 일치

**출력**:
- 🎯 **1순위 (강력 추천) 7개**: 위 3가지 모두 충족
- 🔵 **2순위 (블루오션) 3개**: CTR 높은 니치 태그 (전환 특화)
- 🟡 **대체 후보 7개**: 주등록 무효 시 교체용
- 📦 **복붙용 10개**: 콤마 구분 최종 문자열

---

### STEP 8. 시즌 마케팅 캘린더 (keyword-trend 결합)

**DataLab으로 13개월 월별 트렌드 조회** (`mcp__naver-search__datalab_search`):
- 최대 5개 키워드 그룹 비교
- ratio(상대 검색량, 피크=100) 추출

**추가 분석**:
- 피크 월 (2개월 전이 마케팅 준비 시작 시점)
- 비수기 월
- 시즌 강도 (피크/비수기 비율)

**월별 예산 권장안 테이블** 생성:
| 시기 | 핵심 키워드 | 메시지 | 예산 수준 |
|------|-----------|--------|---------|
| ... | ... | ... | 최고/높음/중간/낮음 |

### STEP 9. HTML 통합 리포트 생성

**저장 경로**: `~/Desktop/team-skills/리포트/YYYY-MM-DD_{상품명}_키워드-최적화.html`

**리포트 구성**:

1. **헤더**: 상품명, 카테고리, 타겟, 분석 일자, KPI (시드/연관/경쟁상품/고객언어/트렌드 개수)
2. **TL;DR**: 핵심 인사이트 5~6개 (그라디언트 카드)
3. **📅 시즌 마케팅 캘린더**: Chart.js 라인 그래프 (월별 ratio) + 월별 액션 캘린더
4. **🏆 키워드 등급표**: A/B/C/D 색상 뱃지 + CTR/경쟁도 하이라이트
5. **📝 스마트스토어 상품명 수정안 3개**: 박스형 표시, 글자수·SEO 체크 포함, 금지 단어 경고
6. **🏷️ 상품태그 10개 추천**: 1순위 7개 + 블루오션 3개 + 복붙용 + 대체 후보 + 유효성 확인 팁
7. **💼 활용 가이드**: 광고 입찰 우선순위, 상세페이지 롱테일, 블로그 SEO 제목
8. **🎯 경쟁사 대비 차별화**: 놓친 키워드 / USP 키워드 / 피해야 할 키워드
9. **✅ 실행 체크리스트**: 9개 액션 아이템

**기술 요구사항**:
- 단일 HTML 파일 (CDN Chart.js만 외부)
- Noto Sans KR 시스템 폰트
- 반응형 레이아웃 (max-width 1100px)
- 프린트 가능 (A4 세로 최적)

## 통합 스킬 특징

이 스킬은 단독으로 `keyword-trend` + `keyword-optimizer` 두 스킬의 역할을 모두 수행한다:

| 영역 | 이 스킬이 제공 |
|------|-------------|
| 어떤 키워드 | ✅ STEP 1~5 (시드 발굴 → 스코어링 → 등급표) |
| 언제 공략 | ✅ STEP 8 (DataLab 월별 트렌드 + 캘린더) |
| 어디에 쓸지 | ✅ STEP 6~7 (상품명 3안 + 태그 10개) |
| 어떻게 시각화 | ✅ STEP 9 (HTML + Chart.js) |

별도 `keyword-trend` 호출은 불필요하지만, 특정 키워드의 시즌성만 빠르게 보고 싶으면 `keyword-trend` 단독 사용도 가능.

## 타 스킬과의 관계

| 스킬 | 역할 | 이 스킬과의 관계 |
|------|------|-----------------|
| keyword-trend | 키워드 계절성/타이밍 단독 분석 | 이 스킬에 **통합됨** (STEP 8) |
| new-product-planner | **신제품** 기획 과정의 키워드 발굴 | 이 스킬은 **기존 상품** 노출 최적화 |
| product-scout | 신상품 **기회 발굴** | 이 스킬은 이미 가진 상품의 노출 최적화 |

## 검증 & 후속 액션

1. **태그 유효성 확인**: 등록 후 네이버 쇼핑에서 "상품명 + 태그" 검색 → 결과 메시지로 판정
2. **상품명 점수**: 네이버 스마트스토어센터 > 상품관리 > 상품명 적합도 점수 확인
3. **A/B 테스트**: 3안 중 2안을 각 1주일씩 등록 → CTR/전환율 비교
4. **정기 업데이트**: 시즌 진입 2개월 전 재분석 권장 (트렌드·경쟁 반영)
