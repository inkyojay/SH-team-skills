# 네이버 키워드 시즌 트렌드 분석

키워드의 검색량과 월별 추이를 결합 분석하여, **언제 마케팅에 집중해야 하는지** 데이터 기반으로 판단하는 스킬.

## 분석 프로세스

### STEP 1. 키워드 검색량 조회 (검색광고 API)

현재 월평균 검색량, 경쟁도, 클릭률 등 정량 데이터를 수집한다.

**실행 방법:**
```bash
python3 ~/.claude/skills/new-product-planner/scripts/naver_keyword_search.py \
  --api-key "$NAVER_API_KEY" \
  --secret-key "$NAVER_SECRET_KEY" \
  --customer-id "$NAVER_CUSTOMER_ID" \
  --keywords "키워드1,키워드2,키워드3" \
  --output results.json
```

**인증 정보 필요:**
- `NAVER_API_KEY`: 네이버 검색광고 API 라이선스 키
- `NAVER_SECRET_KEY`: 네이버 검색광고 API 시크릿 키
- `NAVER_CUSTOMER_ID`: 네이버 검색광고 고객 ID
- 확인 경로: 검색광고 시스템 > API 사용 관리

**주의사항:**
- 키워드에 공백 불가 ("아기 슬리핑백" X -> "아기슬리핑백" O)
- 5개씩 배치 호출, 0.5초 간격

### STEP 2. 월별 트렌드 조회 (DataLab API)

MCP 도구 `mcp__naver-search__datalab_search`를 사용하여 최근 2년간 월별 검색 트렌드를 조회한다.

**호출 파라미터:**
- `startDate`: 2년 전 날짜 (yyyy-mm-dd)
- `endDate`: "today"
- `timeUnit`: "month"
- `keywordGroups`: 분석 대상 키워드 그룹 (최대 5그룹, 그룹당 연관 키워드 포함)

**키워드 그룹 구성 팁:**
- 메인 키워드 + 파생 키워드를 하나의 그룹으로 묶는다
- 예: `{"groupName": "슬리핑백", "keywords": ["슬리핑백", "아기슬리핑백", "유아슬리핑백"]}`
- 최대 5그룹까지 한번에 비교 가능

**응답 데이터:**
- `ratio`: 상대 검색 비율 (전체 기간 중 최고점 = 100)
- 월별 ratio 추이로 시즌 패턴 파악

### STEP 3. 결합 분석

검색량(절대값) + 트렌드(상대 추이)를 결합하여 키워드를 분류한다.

#### 시즌 유형 분류

| 유형 | 판정 기준 | 마케팅 전략 |
|------|----------|------------|
| 강한 계절성 | 피크월/최저월 비율 > 3배 | 피크 2개월 전 콘텐츠 준비, 피크 시 광고 집중 |
| 약한 계절성 | 피크월/최저월 비율 1.5~3배 | 피크 시즌 예산 20~30% 증액 |
| 연중 안정 | 피크월/최저월 비율 < 1.5배 | 상시 SEO + 균등 광고 배분 |

#### 분석 지표

1. **피크 시기**: ratio가 가장 높은 월 (2년 평균)
2. **비수기**: ratio가 가장 낮은 월
3. **시즌 강도**: (피크 평균 ratio) / (비수기 평균 ratio)
4. **상승 진입점**: 비수기에서 피크로 전환되는 월 (마케팅 준비 시점)
5. **YoY 성장률**: 전년 동월 대비 ratio 변화

### STEP 4. 결과 리포트

#### 출력 형식

```
== 키워드 시즌 트렌드 분석 ==

[키워드명]
- 월평균 검색량: N회 (PC: N / Mobile: N)
- 경쟁도: 높음/중간/낮음
- 시즌 유형: 강한 계절성 / 약한 계절성 / 연중 안정
- 피크 시기: N월 (ratio: XX)
- 비수기: N월 (ratio: XX)
- 시즌 강도: X.X배
- 마케팅 준비 시작: N월
- YoY 트렌드: 상승 / 유지 / 하락

[마케팅 캘린더]
1월  ██░░░░░░░░  비수기 - SEO 콘텐츠 축적
2월  ██░░░░░░░░  비수기
...
9월  ██████░░░░  상승기 - 광고 시작
10월 ██████████  피크 - 최대 예산 투입
11월 ████████░░  피크 유지
12월 ██████░░░░  하강기 - 예산 축소
```

## 추가 분석 (MCP 도구 활용)

필요에 따라 아래 MCP 도구로 심화 분석 가능:

| 분석 | MCP 도구 | 용도 |
|------|---------|------|
| 쇼핑 카테고리 트렌드 | `datalab_shopping_category` | 카테고리 전체 시장 추이 |
| 카테고리 내 키워드 비교 | `datalab_shopping_keywords` | 같은 카테고리 내 키워드 경쟁 |
| 성별 트렌드 | `datalab_shopping_by_gender` | 남녀 검색 비율 차이 |
| 연령별 트렌드 | `datalab_shopping_by_age` | 연령대별 관심도 |
| 기기별 트렌드 | `datalab_shopping_by_device` | PC vs 모바일 비중 |

## API 실패 시 대안

1. DataLab MCP 사용 불가 -> `web_search`로 네이버 트렌드 페이지 참조
2. 검색광고 API 사용 불가 -> DataLab ratio만으로 상대적 시즌 패턴 분석
3. 둘 다 불가 -> 사용자에게 네이버 데이터랩 수동 조회 안내
