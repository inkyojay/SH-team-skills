WORKFLOW
전체 워크플로우
6단계의 흐름으로 프로모션 기획서 작성부터 채널별 디자인 병렬 제작까지 완료됩니다.
1
/promotion-plan

프로모션 컨셉 논의

자연어 대화로 프로모션명, 컨셉, 타겟, 채널, 혜택 등 핵심 항목을 하나씩 확정합니다. 이미 알려준 정보는 건너뛰고 부족한 부분만 질문합니다.

2
/promotion-plan

.pen 기획서 + .json 데이터 생성

확정된 내용을 시각적 기획서(.pen)와 구조화된 데이터(.json)로 자동 생성합니다. 기획서는 1680×1188 사이즈의 브랜드 톤 문서입니다.

3
/promotion-design

채널별 디자인 산출물 논의

기획서를 로드한 뒤, 각 채널에서 제작 가능한 포맷을 제시합니다. 사용자가 원하는 디자인을 선택하면 레퍼런스 이미지를 보여줍니다.

4
/promotion-design

레퍼런스 확인 + 소재 수집

기존 레퍼런스 이미지(67장)를 참고하고, 사용자의 새 제품 사진이 있으면 경로를 수집합니다. 없으면 AI로 생성합니다.

5
/promotion-design

병렬 디자인 제작

팀을 생성하고 promotion-designer 에이전트를 산출물 수만큼 병렬 스폰합니다. 각 에이전트가 독립적으로 .pen 파일을 제작합니다.

6
/promotion-design

스크린샷 검토 + 피드백 반영

완성된 디자인의 스크린샷을 보여주고, 수정 요청이 있으면 바로 반영합니다. 모든 디자인이 확정되면 팀을 해체하고 완료합니다.

💡

핵심 포인트

/promotion-plan이 생성한 .json 파일을 /promotion-design이 읽어서 사용합니다. 기획이 확정되어야 디자인 제작이 가능합니다.

SKILLS

/promotion-plan

프로모션 기획

자연어 대화 → .pen 기획서 + .json 데이터

  • 1

    브랜드 컨텍스트 로딩

    brand-guide, products.json 등 썬데이허그 기본 정보를 자동으로 읽어옵니다

  • 2

    대화형 기획

    프로모션명, 컨셉, 톤앤매너, 목적, 타겟, 기간, 채널, 메시지, 제품, 혜택을 논의합니다

  • 3

    확정 요약

    모든 항목을 테이블로 정리하여 사용자 확인을 받습니다

  • 4

    .pen 기획서 생성

    Pencil MCP로 1680×1188 시각적 기획서를 자동 생성합니다

  • 5

    .json 데이터 저장

    구조화된 플랜 데이터를 companion 파일로 저장합니다

  • 6

    핸드오프

    /promotion-design으로 이어가도록 안내합니다

/promotion-design

프로모션 디자인

기획서 로드 → 디자인 논의 → 병렬 .pen 제작

  • 1

    기획서 로딩

    기획서 폴더에서 .json 파일을 찾아 프로모션 데이터를 추출합니다

  • 2

    디자인 논의

    채널별 포맷을 제시하고, 레퍼런스 이미지를 보여주며 소재를 수집합니다

  • 3

    산출물 확정

    최종 디자인 목록을 테이블로 작성하고 사용자 확인을 받습니다

  • 4

    병렬 디자인 실행

    팀 생성 → promotion-designer 에이전트를 병렬 스폰하여 동시에 제작합니다

  • 5

    검토 + 피드백

    스크린샷으로 결과를 확인하고, 수정 사항을 즉시 반영합니다

  • 6

    완료 + 팀 해체

    최종 파일 정리 후 팀을 해체합니다
