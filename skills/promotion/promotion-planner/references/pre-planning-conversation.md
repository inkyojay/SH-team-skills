# 사전 기획 대화 가이드

## 핵심 원칙

1. **바로 실행하지 않는다** — 크롤링, 분석, 전략 수립 모두 대화 후에 시작
2. **하나씩 자연스럽게** — 한꺼번에 4가지를 한번에 묻지 않음
3. **맞장구 + 아이디어 제안** — 사용자 답변에 반응하고, 관련 제안을 곁들인다
4. **모르겠으면 도와준다** — 옵션 제시, 추천, 시즌 정보 알려주기
5. **요약 + 동의** — 4가지 확인 후 정리해서 보여주고 "시작할까요?" 물어보기

---

## 대화 흐름

### 1단계: 기간 + 예산

**시작 멘트 예시**:
- "프로모션 기획 시작할게요! 먼저 기간부터 잡아볼까요?"
- "어떤 기간을 생각하고 계세요?"

**파악할 것**:
- 프로모션 전체 기간 (예: 4~6월, 3개월)
- 마케팅 예산 범위 (구체적 금액 or "체험단 N회, 배포형 N건" 수준)
- 할인 마진 허용 범위 (최대 할인율)

**사용자가 기간만 말한 경우**:
→ 기간에 대한 시즌 인사이트를 짧게 공유 후, 예산으로 자연스럽게 이어가기
예: "4~6월이면 봄→여름 전환기라 소재 교체 수요가 크네요. 예산은 대략 어느 정도로 잡고 계세요?"

**사용자가 예산을 모를 때**:
→ "지난번 기준으로 체험단은 월 몇 회 정도 하셨어요?" 처럼 과거 기준으로 물어보기
→ 또는 "큰 예산 없이 할 수 있는 방식부터 추천해드릴까요?"

---

### 2단계: 집중 제품 / 신제품

**전환 멘트 예시**:
- "이번 기간에 특별히 밀고 싶은 제품이 있나요?"
- "신제품 출시 계획이 있으면 그거에 맞춰 기획하면 좋을 것 같은데요!"

**파악할 것**:
- 이번 기간 집중 제품 (1~3개)
- 신제품 출시 예정 여부 + 예상 출시 시기
- 재고 소진이 필요한 제품 (아울렛 대상 등)
- 시즌 전환 제품 (겨울→봄 전환 등)

**사용자가 모를 때**:
→ "제가 웹사이트에서 현재 제품 상황을 확인해볼게요. 그 다음에 같이 골라볼까요?"
→ 웹 크롤링 후 시즌에 맞는 제품 2~3개를 추천

---

### 3단계: 쇼핑라이브

**전환 멘트 예시**:
- "쇼핑라이브도 계획에 넣을까요?"
- "지난번에 라이브 월 2회로 잡았는데, 이번에도 비슷하게?"

**파악할 것**:
- 라이브 진행 여부 (안 해도 OK)
- 횟수: 월 몇 회?
- 채널: 자사몰 / 네이버 스마트스토어 / 둘 다
- 선호 요일/시간대
- 라이브 전용 혜택 범위 (추가 할인, 사은품 등)

**사용자가 처음이거나 모를 때**:
→ "자사몰 1회 + 스마트스토어 1회로 월 2회 정도가 무리 없이 운영할 수 있는 수준이에요. 이 정도면 어떨까요?"
→ 추천 시간대: "평일 저녁 8시가 육아맘 시청률이 가장 높아요"

---

### 4단계: 특별 이벤트 / 행사

**전환 멘트 예시**:
- "기간 안에 특별히 연계하고 싶은 행사가 있나요?"
- "어린이날처럼 큰 이벤트가 있으면 거기에 맞춰 피크를 잡아볼게요"

**파악할 것**:
- 공휴일/기념일 활용 의향 (어린이날, 가정의 달, 현충일 등)
- 자체 브랜드 이벤트 (론칭, 리뉴얼, 기념일 등)
- 외부 협업/콜라보 일정
- 업계 행사 (베이비페어 등)

**사용자가 모를 때**:
→ 해당 기간의 주요 이벤트를 알려주고 선택하게 한다
예: "4~6월이면 어린이날(5/5), 가정의 달, 여름 시즌 오픈이 있어요. 어린이날은 번들 세트 특가에 딱이고, 가정의 달은 멤버십 이벤트로 좋아요. 어떤 것들을 활용해볼까요?"

---

## 대화 종료 → 기획 요약

4가지 항목이 모두 확인되면 아래 형태로 정리:

```
📋 기획 방향 정리

• 기간: 4~6월 (3개월)
• 예산: 체험단 월 1회, 배포형 월 2~3건, 할인 최대 30%
• 집중 제품: ABC 침대 (4월 말 출시), 메쉬 슬리핑백 (6월 주력)
• 쇼핑라이브: 월 2회 (자사몰 1 + 스마트스토어 1), 목요일 20시
• 행사 연계: 어린이날 플래시세일, 가정의 달 멤버십, 여름 시즌 UGC

이 방향으로 기획 시작할까요?
```

**동의를 받은 후에만** STEP 1(웹사이트 크롤링)으로 진행한다.

## 특수 상황 처리

### 사용자가 이미 구체적 요청을 한 경우
예: "4~6월 행사랑 쇼핑라이브 월 2회 넣어서 플랜 짜줘"
→ 이미 일부 정보가 있으므로, 확인된 항목은 건너뛰고 부족한 항목만 물어본다
→ "기간이랑 라이브 계획은 확인했어요! 집중할 제품이나 예산 범위도 알려주시면 바로 기획할게요"

### 사용자가 "그냥 알아서 해줘"라고 할 때
→ "그러면 제가 웹사이트 확인하고 시즌에 맞게 제안 드릴게요! 혹시 예산 범위만 대략 알려주실 수 있나요?"
→ 최소한의 정보만 확인 후 나머지는 자동으로 채워서 진행

### 이전 플랜을 HTML로만 정리하는 경우
→ STEP 0 대화를 생략하고 바로 HTML 생성으로 간다
→ "이전에 기획한 내용을 HTML로 정리해드릴게요!"
