> 일 예산: 200,000원 (월 약 600만원)

## Part 1. 두 영상 핵심 통합 요약

### 구식 조언 vs 실제로 해야 할 것

| 구식 조언 (STOP) | 새로운 방식 (DO) |
|------------------|------------------|
| 크리에이티브 적게, 각각 다르게 | **핵심 컨셉의 대량 이터레이션** |
| 학습 단계(Learning Phase) 주의 | **무시하고 공격적 스케일링** |
| 매일 같은 금액 지출 | **요일별 전환 데이터에 맞춰 차별 배분** |
| 타겟 오디언스 → 광고 제작 | **광고 컨셉 → 알고리즘이 오디언스 매칭** |
| ROAS 높은 광고만 밀기 | **수익성 있으면 전부 운영 (스케일 우선)** |

### Creative Flywheel (크리에이티브 플라이휠)

```
경쟁사 Ad Library 분석
        ↓
  위닝 컨셉 발굴
        ↓
  자사 제품으로 재해석 + 이터레이션 제작
        ↓
  Prospecting CBO 캠페인에 런칭
  (Ad Set Minimum = 1x CPA)
        ↓
  7일 대기 → 데이터 수집
        ↓
  Incremental Attribution으로 분석
        ↓
  위닝 광고 → 추가 이터레이션 제작
        ↓
  (다시 처음으로 → 반복)
```

### M3 프레임워크

| 단계 | 내용 | SUNDAY HUG 적용 |
|------|------|-----------------|
| **M1: 계정 구조** | Prospecting / Retargeting / Retention 분리 | **즉시 적용** |
| **M2: 고급 분석** | 플랫폼, 게재위치, 연령, 성별, 요일 분석 | **즉시 적용** |
| **M3: 비용 관리** | Cost Cap으로 CPA/ROAS 안정화 | 월 1천만원+ 이후 적용 |

## Part 3. 캠페인 구조 설계

### 전체 계정 구조 (M1 적용)

```
SUNDAY HUG Ad Account
│
├─ [CBO] 1. Prospecting (신규 고객) ──── 일 140,000원 (70%)
│   ├─ Ad Set: 꿀잠팩 01 (슬리핑백 컨셉 A)
│   ├─ Ad Set: 꿀잠팩 02 (스와들 컨셉 A)
│   ├─ Ad Set: 꿀잠팩 03 (세트 상품 컨셉 A)
│   ├─ Ad Set: 꿀잠팩 04 (UGC/리뷰 컨셉)
│   ├─ Ad Set: 꿀잠팩 05 (신규 테스트)
│   └─ ... (매주 새 Ad Set 추가)
│
├─ [ABO] 2. Retargeting (관심 고객) ──── 일 40,000원 (20%)
│   ├─ Ad Set: 사이트 방문자 7일
│   ├─ Ad Set: 장바구니 이탈자 14일
│   └─ Ad Set: 인스타 인게이지먼트 30일
│
└─ [ABO] 3. Retention (기존 고객) ──── 일 20,000원 (10%)
    ├─ Ad Set: 구매자 크로스셀 (슬리핑백→스와들)
    └─ Ad Set: 구매자 업셀 (단품→세트)
```

### 예산 배분 근거

| 캠페인 | 일 예산 | 비율 | 역할 |
|--------|---------|------|------|
| Prospecting | 140,000원 | 70% | 신규 고객 유입 (성장 엔진) |
| Retargeting | 40,000원 | 20% | 구매 전환 극대화 |
| Retention | 20,000원 | 10% | LTV 극대화, 재구매 유도 |

> **핵심:** Facebook에 예산 통제를 맡기지 말 것. Prospecting에 충분한 예산 확보.

## Part 4. Prospecting CBO 운영법 (Creative Flywheel)

### Ad Set 런칭 규칙

| 항목 | 설정 |
|------|------|
| 캠페인 유형 | CBO (Campaign Budget Optimization) |
| 일 캠페인 예산 | 140,000원 |
| Ad Set당 크리에이티브 | 4~6개 |
| Ad Set Minimum Budget | **1x CPA 타겟** (예: 25,000원) |
| 테스트 기간 | 7일 |
| 타겟팅 | Broad (한국, 여성, 25~44세) |
| 게재위치 | Advantage+ (자동), 이후 M2 분석으로 최적화 |
| 전환 목표 | Purchase |

### 주간 운영 사이클

```
월요일: 지난주 데이터 분석 (Incremental Attribution)
        → 위닝 광고 확인, 루저 Ad Set Minimum 제거

화요일: 위닝 광고 기반 이터레이션 기획
        → 같은 컨셉, 다른 제품/이미지/카피

수~목요일: 새 크리에이티브 제작 (4~6개)

금요일: 새 Ad Set 런칭 + Ad Set Minimum 설정
        → 주말 높은 트래픽 활용

토~일요일: 데이터 축적 (건드리지 않기)
```

### 7일 판단 기준

| 상태 | ROAS | 조치 |
|------|------|------|
| Winner | 목표 ROAS 이상 | Minimum 유지 또는 제거 (자연 스케일) |
| Potential | 목표의 70~100% | Minimum 제거 후 1주 더 관찰 |
| Loser | 목표의 70% 미만 | Minimum 제거 (자연 소멸) |

## Part 5. 크리에이티브 전략

### Step 1: 경쟁사 컨셉 소싱

**벤치마킹 대상 브랜드** (자사보다 한두 단계 위):

| 브랜드 | 카테고리 | Ad Library 검색어 |
|--------|----------|-------------------|
| 에르고베이비 코리아 | 아기띠/수면 | Ergobaby Korea |
| 코니바이에린 | 아기띠 | Konny by Erin |
| 라라스컴퍼니 | 속싸개/수면 | LaLa's Company |
| 보니숲 | 유아 의류 | Bonniesoup |
| 아기바다 | 유아 수면용품 | - |
| Love to Dream | 스와들 (해외) | Love to Dream |
| Kyte Baby | 슬리핑백 (해외) | Kyte Baby |
| Nested Bean | 수면 솔루션 (해외) | Nested Bean |

**Ad Library 분석 방법:**
1. [Meta Ad Library](https://www.facebook.com/ads/library) 접속
2. 위 브랜드 검색 → 가장 오래 운영 중인 광고 = 위닝 광고
3. 컨셉, 포맷, 카피 패턴 기록
4. SUNDAY HUG 제품으로 재해석

### Step 2: 핵심 크리에이티브 컨셉 6가지

#### 컨셉 1: "비포 & 애프터" (수면 변화)

```
[Before] 밤새 깨는 아기, 지친 엄마 표정
    ↓ 전환 효과
[After] 꿀잠 자는 아기, 미소 짓는 엄마

카피: "매일 밤 전쟁이었던 수면... 꿀잠 슬리핑백 하나로 달라졌어요"
CTA: 지금 구매하기 → sundayhug.kr
```

**이터레이션:**
- 제품별 (슬리핑백 / 스와들 / 아기침대)
- 모델별 (다른 아기, 다른 엄마)
- 카피 변형 ("3시간 → 8시간 연속 수면")

#### 컨셉 2: "리뷰/후기 카드" (사회적 증거)

```
실제 구매 후기 텍스트 + 별점
제품 사용 사진
"리뷰 3,000개 돌파" 배지

카피: "모로반사 때문에 매번 깨던 아기가 스와들 스트랩 하나로..."
```

**이터레이션:**
- 후기별 (다른 리뷰 텍스트)
- 제품별 (슬리핑백 후기 / 스와들 후기)
- 레이아웃 변형 (카드형 / 스크린샷형)

#### 컨셉 3: "제품 데모" (기능 시연)

```
15~30초 영상:
- 슬리핑백 입히는 과정 (3초 컷)
- 지퍼/단추 디테일 클로즈업
- 아기가 편안하게 잠든 모습

카피: "버튼 하나로 3초 만에. 꿀잠 슬리핑백"
```

**이터레이션:**
- 제품별 시연 (밤부 / 트리플 / 윈터)
- 포커스 변형 (소재 / 착용법 / 사이즈)

#### 컨셉 4: "엄마의 하루" (라이프스타일)

```
브이로그 스타일 15초:
- 아침에 아기 깨우기
- 슬리핑백 벗기기
- 수유 → 낮잠 → 스와들 입히기
- "오늘도 꿀잠 성공"

카피: "6개월차 엄마의 수면 루틴. 비결은 딱 하나."
```

#### 컨셉 5: "세트 구성 비주얼" (세트 상품 전용)

```
3종 세트 구성품을 예쁘게 배치한 플랫레이
각 제품 역할 텍스트 오버레이:
- 스와들 A: "신생아기 꿀잠"
- 스와들 B: "성장기 꿀잠"
- 백색소음기: "수면 환경 완성"

카피: "출산 준비, 이 세트 하나면 끝. 꿀잠 스타터 3종"
할인 강조: "개별 구매 대비 XX% 할인"
```

#### 컨셉 6: "성장 단계별 솔루션" (교육형)

```
캐러셀 광고:
슬라이드 1: "0~3개월: 스와들 스트랩으로 모로반사 방지"
슬라이드 2: "3~6개월: 스와들 포켓으로 자연스러운 전환"
슬라이드 3: "6~24개월: 꿀잠 슬리핑백으로 숙면"
슬라이드 4: "우리 아기 단계에 맞는 꿀잠 찾기 →"

카피: "아기는 자라고, 꿀잠은 계속됩니다."
```

### Step 3: 포맷 믹스

| 포맷 | 비율 | 용도 |
|------|------|------|
| **정적 이미지** | 40% | 제품샷, 리뷰 카드, 플랫레이 |

## Part 9. 런칭 로드맵

### Week 1: 기반 세팅

- [ ] Meta Pixel / Conversions API 정상 작동 확인
- [ ] 커스텀 오디언스 생성 (구매자, 방문자, 장바구니, IG 인게이지먼트)
- [ ] 경쟁사 Ad Library 분석 (최소 5개 브랜드, 브랜드당 위닝 컨셉 3개)
- [ ] 계정 구조 세팅 (Prospecting CBO + Retargeting ABO + Retention ABO)
- [ ] 첫 크리에이티브 제작: 컨셉 1~3에서 각 4~6개 = 총 12~18개

### Week 2: 첫 런칭

- [ ] Prospecting CBO에 Ad Set 3개 런칭 (컨셉별 1개씩)
- [ ] Ad Set Minimum = 25,000원 (1x CPA 타겟) 설정
- [ ] Retargeting 캠페인 런칭 (리뷰 카드 + 제품 데모)
- [ ] 7일 대기 (건드리지 않기!)

### Week 3: 첫 분석 + 이터레이션

- [ ] Incremental Attribution으로 위닝 광고 확인
- [ ] 루저 Ad Set Minimum 제거
- [ ] 위닝 컨셉 기반 이터레이션 4~6개 제작
- [ ] 새 Ad Set 런칭

### Week 4: 플라이휠 가동

## Part 10. 크리에이티브 제작 체크리스트

### 이미지 광고

- [ ] 1080x1080 (피드) + 1080x1920 (스토리/릴스) 두 사이즈 제작
- [ ] 제품이 전체 면적의 50% 이상 차지
- [ ] 텍스트 20% 이하 (Meta 권장)
- [ ] 3초 내 제품/혜택 인식 가능
- [ ] 브랜드 로고 작게 포함 (좌상단 또는 우하단)

### 영상 광고 (릴스)

- [ ] 9:16 비율, 15~30초
- [ ] 처음 3초 훅 (문제 제시 or 비주얼 임팩트)
- [ ] 캡션(자막) 필수 (무음 시청 80%)
- [ ] 세이프존 준수 (상단 108px, 하단 320px 여백)
- [ ] 사람 등장 권장 (CTR +27%)
- [ ] CTA 명확: "링크 클릭" or "프로필에서 구매"

### 이터레이션 제작 규칙

하나의 컨셉에서 변형할 수 있는 요소:

| 변형 요소 | 예시 |
|-----------|------|
| 제품 | 슬리핑백 밤부 → 트리플 → 윈터 |
| 이미지/모델 | 다른 아기, 다른 엄마 |
| 카피 헤드라인 | "꿀잠의 비밀" → "밤새 깨지 않는 비결" |
| 배경색/레이아웃 | 화이트 → 베이지 → 연핑크 |
| CTA | "지금 구매" → "무료배송 혜택 받기" |
| 가격/프로모션 | 정가 → 할인가 → 세트 할인 |

> **Sam Piliero 핵심:** 완전히 다른 광고를 만들지 말 것. 같은 템플릿에 요소만 바꿔서 대량 생산.

## 핵심 요약

```
1. 계정 구조: Prospecting 70% / Retargeting 20% / Retention 10%
2. 크리에이티브: 6가지 컨셉 × 제품별/모델별 이터레이션 = 대량 생산
3. 테스트: CBO + Ad Set Minimum(1x CPA) + 7일 대기
4. 분석: Incremental Attribution으로 위닝 확인
5. 반복: 위닝 컨셉 → 이터레이션 → 테스트 → 분석 → 반복
6. 최적화: 요일/연령/게재위치 Breakdown으로 예산 효율화
```

> **기억할 것:** 200,000원/일 예산에서는 M1(계정 구조)과 M2(분석)에 집중.
> M3(Cost Cap)은 월 1천만원+ 도달 후 적용.
> 학습 단계는 무시. ROAS가 유지되면 공격적으로 스케일.
