# 신제품 무드보드 생성 가이드

## 목적

신제품 기획 과정에서 수집한 이미지/컬러/키워드를 조합하여 **시각적 무드보드**를 자동 생성한다.
경영진 보고, 디자인팀 방향 공유, 생산팀 커뮤니케이션에 "이런 느낌의 제품을 만들려 한다"를 한 장으로 전달.

## 무드보드 소재 수집 (PHASE 2~3에서 자동 진행)

### 이미지 소스 4가지

| 소스 | 수집 방법 | 수집 시점 | 예시 |
|------|---------|---------|------|
| 경쟁사 제품 사진 | PHASE 2.5 쇼핑 API → image 필드 | PHASE 2.5 | 포몽드 에떼쿨 제품 이미지 |
| 디자인 레퍼런스 | PHASE 3 image_search 도구 | PHASE 3 | "baby cooling mat premium design" |
| 해외 레퍼런스 | PHASE 3 image_search 도구 | PHASE 3 | "infant summer bedding aesthetic" |
| 사용자 업로드 | 직접 업로드 | 아무 때나 | 참고하고 싶은 디자인/소재 사진 |

### 컬러 팔레트 추출

무드보드에 포함할 컬러는 3가지 소스에서 수집:
1. **브랜드 기존 컬러**: 썬데이허그 메인 컬러 (#1D9E75 등)
2. **경쟁사 트렌드 컬러**: 경쟁사 제품/상세페이지에서 추출
3. **시즌 컬러**: 여름이면 쿨톤(블루/민트/화이트), 겨울이면 웜톤

### 키워드 추출

무드보드에 들어갈 무드 키워드:
- PHASE 8 소구점에서 추출: "쿨링", "프리미엄", "안심", "꿀잠"
- PHASE 2.6 VOC에서 추출: 고객이 자주 쓰는 감성 단어
- 사용자가 직접 지정: "이런 느낌이었으면 좋겠어"

## 무드보드 생성 방법

### 방법 1: Python 스크립트 (PNG 이미지 생성)

`scripts/generate_moodboard.py` 활용:

```bash
python generate_moodboard.py \
  --images /tmp/product_images/ \
  --colors "#1D9E75,#2563EB,#F0F9FF,#1E293B,#F59E0B" \
  --keywords "쿨링,프리미엄,아기꿀잠,국내산,안심수면" \
  --brand "썬데이허그 쿨슬립" \
  --template presentation \
  --output /mnt/user-data/outputs/coolsleep-moodboard.png
```

템플릿 옵션:
- `grid`: 3×2 그리드 + 사이드바 (기본, 레퍼런스 비교용)
- `presentation`: 메인 이미지 크게 + 보조 이미지 + 패널 (보고용)

### 방법 2: HTML 인터랙티브 무드보드 (리포트 내장)

HTML 리포트의 "레퍼런스 보드" 탭을 무드보드 형태로 구성:
- 이미지 그리드 + 컬러 팔레트 + 키워드 클라우드
- 카테고리별 필터 (경쟁사/해외/디자인/사용자업로드)
- 이미지 클릭 시 확대 + 출처 표시

### 방법 3: 대화 중 즉석 생성

PHASE 3 진행 중 image_search로 이미지를 보여주면서:
"이런 느낌의 제품을 참고하면 좋을 것 같아요. 마음에 드는 방향이 있으세요?"
→ 사용자 피드백 반영하여 무드보드 방향 조율

## 무드보드 구성 요소

### 최종 무드보드 레이아웃

```
┌──────────────────────────────────────────┐
│  [브랜드명] 신제품 무드보드               │
│  [제품명 (가칭)] — [컨셉 한 줄]          │
├──────────────────────┬───────────────────┤
│                      │ 컬러 팔레트       │
│   메인 이미지        │ ■ ■ ■ ■ ■       │
│   (제품 방향성)      │                   │
│                      │ 무드 키워드       │
├──────────┬───────────┤ · 쿨링            │
│ 레퍼런스1│ 레퍼런스2 │ · 프리미엄         │
│ (소재)   │ (디자인)  │ · 아기꿀잠         │
├──────────┼───────────┤ · 국내산           │
│ 경쟁사1  │ 경쟁사2   │ · 안심수면         │
│ (참고)   │ (차별화)  │                   │
└──────────┴───────────┴───────────────────┘
```

### 이미지 배치 권장

| 위치 | 이미지 유형 | 목적 |
|------|-----------|------|
| 메인 (가장 큰 자리) | 가장 이상적인 레퍼런스 | "이런 느낌으로 만들자" |
| 상단 2장 | 소재/디자인 레퍼런스 | 디테일 방향성 |
| 하단 2장 | 경쟁사 제품 | "이것보다 나은 것을 만들자" |
| 사이드바 | 컬러 + 키워드 | 전체 무드/톤 |

## 무드보드 생성 타이밍

| 시점 | 버전 | 용도 |
|------|------|------|
| PHASE 3 중간 | 초안 무드보드 | 사용자와 방향 확인 ("이 느낌 맞나요?") |
| PHASE 3 완료 | 1차 무드보드 | 레퍼런스 이미지 + 경쟁사 제품 포함 |
| PHASE 6 컨셉 확정 | 최종 무드보드 | 컬러+키워드+소구점 반영, 리포트에 포함 |
| PHASE 8 소구점 이후 | 마케팅 무드보드 | 상세페이지/SNS 방향성용 (필요 시) |

## 사용자 대화 흐름

**PHASE 3 시작 시:**
"경쟁사 제품과 레퍼런스 이미지를 수집하면서 신제품 무드보드도 함께 만들어볼게요. 혹시 미리 참고하고 싶은 이미지나 느낌이 있으면 업로드해주세요."

**이미지 수집 후:**
"이런 레퍼런스들을 모았는데요, 이 중에서 우리 제품 방향과 가장 가까운 느낌은 어떤 건가요?"

**무드보드 생성 후:**
"무드보드를 만들었어요. 전체적인 톤이나 수정할 부분 있으면 말씀해주세요."

## 리포트 반영

HTML 리포트에 무드보드를 2가지 형태로 포함:
1. **레퍼런스 보드 탭**: 인터랙티브 이미지 갤러리 + 컬러/키워드 (HTML)
2. **무드보드 이미지**: generate_moodboard.py로 생성한 PNG를 다운로드 가능하게 제공

무드보드 PNG는 별도 파일로도 outputs에 저장하여 사용자가 바로 팀에 공유할 수 있게.
