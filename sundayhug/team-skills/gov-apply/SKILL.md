# 지원사업 신청서 작성 스킬 (gov-apply) v3

## Overview

정부/공공기관 지원사업 신청서를 **사용자와 대화하며** 작성하는 통합 스킬.
평가표를 역분석하여 선정 확률을 높이고, 기업정보 자동매핑 + 대화형 전략 수립으로 고품질 서류를 생성합니다.

## When to Use

- "지원사업 신청서 작성해줘"
- "사업계획서 써줘"
- "참여신청서 채워줘"
- "공고문 분석하고 서류 작성해줘"
- "지원사업 서류 준비"
- "/gov-apply {사업폴더명}"

## Workflow (8 Phase)

### Phase 0: 양식 확보 (사용자 참여) ★ v3.1

**HWP 파일은 표 구조를 정확히 읽을 수 없으므로, PDF 변환 후 분석합니다.**

사용자에게 안내:
```
이 지원사업의 공고문/신청서 양식이 HWP로 되어있습니다.
정확한 표 구조 파악을 위해 PDF로 변환해서 제공해주시면
동일한 양식으로 문서를 작성해드리겠습니다.

변환 방법:
1. HWP 파일을 한글(아래아 한글)로 열기
2. 파일 → 다른 이름으로 저장 → PDF 선택
3. 저장된 PDF를 해당 사업 폴더에 넣어주세요

필요한 파일:
☐ 공고문 → PDF
☐ 신청서/참여계획서 양식 → PDF (빈 양식 그대로)
```

**왜 PDF인가?**
- PDF는 Read 도구로 표 구조/칸 배치/항목명을 시각적으로 정확히 파악 가능
- HWP의 hwp5txt는 텍스트만 추출되어 표 구조가 깨짐
- PDF 양식을 보고 python-docx로 동일한 표 구조를 DOCX로 재현

**파일 네이밍 규칙:**
```
{사업폴더}/
├── 공고문.pdf          ← 사용자가 HWP→PDF 변환
├── 신청서양식.pdf       ← 빈 양식 PDF (표 구조 파악용)
├── 공고문.hwp          ← 원본 (보관용)
└── 신청서양식.hwp       ← 원본 (보관용)
```

### Phase 1: 공고문 + 양식 분석 (자동)

1. **공고문 PDF 읽기** → Read 도구로 텍스트+표 구조 추출
2. **신청서 양식 PDF 읽기** → 표의 행/열 구조, 각 칸의 라벨(항목명), 칸 크기 파악
3. 추출 항목:
   - 사업명, 사업기간, 신청기간
   - 필수 서류 목록 (별지/붙임 번호, 온라인제출 vs 파일업로드 구분)
   - 신청자격 요건
   - 지원제외 대상
   - **양식 구조** (표 레이아웃, 필드명, 글자수 제한 등) ← PDF에서 정확히 파악
   - **평가표** (배점, 항목별 기준) ← Phase 1A로 분리
   - **가점 항목** (정책 가점 등)
4. 출력: 구조화된 분석 결과를 사용자에게 보고

## 문서 유형별 템플릿 ★ NEW

### 유형1: 참여계획서 / 사업수행계획서
**패턴**: 현황 → 목표 → 전략 → 일정 → 예산
**핵심**: 구체적 수치 + 단계별 실행계획
**주의**: 글자수 제한 빈번 (200자, 500자 등)

### 유형2: 참가신청서 (전시회/유통망)
**패턴**: 기업정보 → 제품정보 → 참가목적 → 바이어 타겟
**핵심**: 제품 경쟁력 + 해당 시장 적합성
**주의**: 전시품목 상세서 별도 작성 필요

### 유형3: 바우처 신청서 (데이터/클라우드/S바우처)
**패턴**: 기업정보 → 활용계획 → 기대효과 → 예산
**핵심**: 바우처 활용의 구체적 계획 + ROI
**주의**: 수요기업+공급기업 컨소시엄 구조인 경우 있음

### 유형4: 동의서/서약서/확약서
**패턴**: 기업정보 채움 + 체크 + 서명
**핵심**: 기계적 작성 가능 (자동화 우선)
**주의**: 온라인 제출인지 파일 업로드인지 구분

## 품질 검증 규칙 ★ NEW

### 글자수 검증
```python
def validate_char_count(text, limit, section_name):
    count = len(text.replace(' ', '').replace('\n', ''))
    if count > limit:
        return f"⚠️ {section_name}: {count}자 ({limit}자 초과 {count-limit}자)"
    elif count < limit * 0.7:
        return f"💡 {section_name}: {count}자 (여유 있음, 내용 보강 가능)"
    else:
        return f"✅ {section_name}: {count}자 (적정)"
```

### 필수항목 체크
- 회사명, 대표자, 사업자번호: 기업정보.json과 일치 여부
- 금액/수치: 재무제표/수출실적과 일치 여부
- 날짜: 신청기간 내 작성일 여부

### 평가표 기반 자기 채점
- 각 평가항목별 예상 점수 산출
- 총점 예상 + 보완 포인트 제시
- "이 부분을 이렇게 수정하면 +N점 가능" 제안

## 병렬 팀 실행 인터페이스 ★ NEW

### 에이전트 팀 입력 포맷
```
/gov-apply {사업폴더명}

입력 파일:
- 기업정보.json       (필수)
- 전략방향.json       (필수 - 대화에서 생성)
- {사업폴더}/공고문    (필수 - PDF/HWP)

출력 파일:
```

1. 전략방향.json 로드 → 사용자 대화 없이 자동 적용

3. 문서 유형 자동 판별 → 해당 템플릿 적용
4. 기업정보 + 전략방향 기반 초안 자동 생성
5. 품질 검증 (글자수, 필수항목, 자기채점)
6. 제출 체크리스트 생성

## 문서 작성 기술

### HWP 읽기
```bash
hwp5txt input.hwp    # HWP 5.x 텍스트 추출
```

### DOCX 생성 (python-docx)
```python
from docx import Document
from docx.shared import Pt, Cm
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.enum.table import WD_TABLE_ALIGNMENT

doc = Document()
# 제목, 표, 본문 작성
doc.save('output.docx')
```

### HWPX 변환
```bash
pypandoc-hwpx input.docx -o output.hwpx
```

### XLSX 편집 (openpyxl)
```python
from openpyxl import load_workbook
wb = load_workbook('template.xlsx')
ws = wb.active
ws['B2'] = company_info['name']
wb.save('filled.xlsx')
```

## 작성 원칙

1. **평가표 역맞춤**: 평가항목별 배점을 확인하고, 높은 배점 항목에 집중
2. **대화 우선**: 전략적 판단은 반드시 사용자와 대화 후 결정
3. **구체적 수치**: "매출 증가" → "매출 US$138,816에서 US$200,000으로 44% 성장"
4. **증빙 연동**: 작성 내용과 증빙 서류의 수치가 반드시 일치
5. **글자수 준수**: 제한이 있으면 반드시 검증 후 제출
6. **일관성**: 같은 사업 내 모든 서류에서 동일한 정보 사용
7. **재사용**: 대화에서 합의된 방향성(전략방향.json) → 다른 지원사업에 적용
