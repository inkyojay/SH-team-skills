# 상품 키워드 최적화 스킬

기존 상품의 네이버 쇼핑 노출을 극대화하기 위해, 데이터 기반으로 최적의 키워드 조합을 발굴하는 스킬.

## 핵심 원칙

- **블루 키워드**: 검색량은 충분하고, 경쟁은 낮은 키워드를 찾는다
- **고객 언어**: 판매자 용어가 아닌, 고객이 실제 검색하는 표현을 쓴다
- **데이터 우선**: 감이 아닌 검색량/경쟁도/클릭률 수치로 판단한다

## 워크플로우

사용자에게 먼저 다음을 확인한다:
1. **상품명** (또는 상품 카테고리)
2. **현재 사용 중인 키워드** (있으면)
3. **타겟 고객** (연령, 성별 등)

---

### STEP 1. 시드 키워드 수집

상품 정보로부터 초기 키워드 후보를 만든다.

#### 1-1. 상품 속성 분해

상품을 속성별로 분해한다:
- **카테고리**: 슬리핑백, 보행기, 젖병 등
- **소재/특성**: 오가닉, 메쉬, 실리콘, 항균 등
- **타겟**: 신생아, 돌아기, 임산부 등
- **시즌/용도**: 여름용, 사계절, 외출용, 실내용 등
- **브랜드 키워드**: 썬데이허그 등

조합 공식:
```
[타겟] + [소재/특성] + [카테고리] + [시즌/용도]
→ "신생아 오가닉 슬리핑백 여름용"
```

최소 **15개** 시드 키워드를 생성한다.

#### 1-2. 네이버 쇼핑 경쟁 상품 역추적

MCP 도구 `mcp__naver-search__search_shop`으로 메인 키워드 검색 후, 상위 상품들의:
- 상품명에 포함된 키워드 추출
- 카테고리 태그 확인
- 반복 등장하는 표현 수집

```
예: "슬리핑백" 검색 → 상위 10개 상품명에서 공통 키워드 추출
→ "사계절", "겉싸개", "신생아", "퇴원", "속싸개" 등 발견
```

#### 1-3. 고객 언어 수집

실제 고객이 사용하는 표현을 수집한다:

- `mcp__naver-search__search_kin`: 지식인에서 "[카테고리] 추천" 검색 → 질문 제목의 표현 수집
- `mcp__naver-search__search_cafearticle`: 맘카페에서 "[카테고리] 어떤거" 검색 → 실사용 표현 수집
- `mcp__naver-search__search_blog`: 블로그에서 "[카테고리] 비교" 검색 → 리뷰어 표현 수집

수집 포인트:
- 질문 제목의 자연어 표현 (예: "신생아 잘때 뭐 입혀요" → "신생아 수면 옷")
- 브랜드명 없이 검색하는 패턴
- "~추천", "~비교", "~차이", "~후기" 패턴

---

### STEP 2. 검색량 + 경쟁도 조회

수집한 키워드 후보를 네이버 검색광고 API로 정량 분석한다.

#### 실행 방법

```bash
python3 ~/Projects/team-skills/skills/new-product-planner/scripts/naver_keyword_search.py \
  --api-key "$NAVER_API_KEY" \
  --secret-key "$NAVER_SECRET_KEY" \
  --customer-id "$NAVER_CUSTOMER_ID" \
  --keywords "키워드1,키워드2,키워드3,키워드4,키워드5" \
  --output /tmp/keyword_result.json
```

API 키가 환경변수에 없으면 사용자에게 요청한다:
> "네이버 검색광고 API 키가 필요합니다. 검색광고 시스템 > API 사용 관리에서 API 키, 시크릿 키, 고객 ID를 확인해주세요."

#### 수집 데이터

| 항목 | 설명 | 활용 |
|------|------|------|
| monthlyPcQcCnt | 월간 PC 검색량 | 검색 볼륨 |
| monthlyMobileQcCnt | 월간 모바일 검색량 | 검색 볼륨 (주력) |
| monthlyAveMobileClkCnt | 월간 모바일 클릭수 | 실제 유입 가능성 |
| monthlyAveMobileCtr | 모바일 클릭률 | 구매 의도 판단 |
| compIdx | 경쟁 정도 | 높음/중간/낮음 |
| plAvgDepth | 평균 광고 노출수 | 광고 경쟁 강도 |
| relKeyword | 연관 키워드 | 추가 후보 발굴 |

**중요**: API가 반환하는 `relKeyword` (연관 키워드)도 추가 후보로 수집한다. 이것이 핵심 확장 소스다.

---

### STEP 3. 쇼핑 트렌드 분석 (DataLab)

MCP 도구로 키워드의 쇼핑 트렌드를 분석한다.

#### 3-1. 카테고리 코드 찾기

`mcp__naver-search__find_category`로 상품 카테고리 코드를 먼저 조회한다.

#### 3-2. 키워드 비교

`mcp__naver-search__datalab_shopping_keywords`로 후보 키워드 간 쇼핑 트렌드를 비교한다.

```
예: "슬리핑백" vs "아기슬리핑백" vs "신생아슬리핑백"
→ 최근 6개월 트렌드 비교
```

#### 3-3. 타겟 분석 (선택)

필요시 성별/연령/디바이스별 분석:
- `datalab_shopping_keyword_by_gender`: 성별 검색 비중
- `datalab_shopping_keyword_by_age`: 연령대별 검색 비중
- `datalab_shopping_keyword_by_device`: PC vs 모바일 비중

---

### STEP 4. 키워드 스코어링

수집한 데이터로 각 키워드에 점수를 매긴다.

#### 스코어링 공식

```
키워드 점수 = (월간 모바일 검색량 × 모바일 클릭률) / 경쟁 지수
```

경쟁 지수 변환:
- 높음 = 3
- 중간 = 2
- 낮음 = 1

#### 분류 기준

| 등급 | 조건 | 활용 |
|------|------|------|
| S (블루 키워드) | 검색량 1000+ & 경쟁 낮음 | 최우선 공략 |
| A (핵심 키워드) | 검색량 5000+ & 경쟁 중간 | 메인 키워드 |
| B (보조 키워드) | 검색량 500~5000 & 경쟁 중간 이하 | 서브 키워드 |
| C (롱테일) | 검색량 100~500 & 경쟁 낮음 | 상세설명/태그 활용 |
| D (비추천) | 검색량 100 미만 또는 경쟁 높음 & 검색량 낮음 | 제외 |

---

### STEP 5. 최종 키워드 추천

결과를 다음 형태로 정리하여 사용자에게 제공한다:

#### 출력 포맷

```
========================================
 [상품명] 키워드 최적화 리포트
========================================

1. 추천 상품명 조합 (상위 3안)
   ① [메인키워드] [서브1] [서브2] [브랜드] [상품명]
   ② ...
   ③ ...

2. 키워드 등급표
   ┌────────┬──────────┬────────┬──────┬──────┐
   │ 등급   │ 키워드    │ 월검색량 │ 경쟁  │ 점수  │
   ├────────┼──────────┼────────┼──────┼──────┤
   │ S      │ ...      │ ...    │ 낮음  │ ...  │
   │ A      │ ...      │ ...    │ 중간  │ ...  │
   └────────┴──────────┴────────┴──────┴──────┘

3. 활용 가이드
   - 상품명: S+A 등급 키워드 조합
   - 태그: B+C 등급 키워드
   - 상세페이지: C 등급 롱테일 키워드를 자연스럽게 포함
   - 광고: A 등급 중 CTR 높은 키워드 우선 입찰

4. 경쟁 상품 대비 차별화 키워드
   - 경쟁사가 놓치고 있는 키워드
   - 우리만 쓸 수 있는 USP 키워드

5. 타겟 인사이트 (DataLab 기반)
   - 주요 검색 성별/연령
   - 모바일 vs PC 비중
   - 최근 트렌드 방향 (상승/하락/유지)
```

## 기존 스킬과의 관계

| 스킬 | 역할 | 이 스킬과의 차이 |
|------|------|----------------|
| keyword-trend | 키워드의 **계절성/타이밍** 분석 | 이 스킬은 **어떤 키워드**를 쓸지 결정 |
| new-product-planner | **신제품** 기획 과정의 키워드 발굴 | 이 스킬은 **기존 상품** 노출 최적화 |

필요시 keyword-trend와 조합하여 "어떤 키워드를 + 언제" 공략할지 통합 전략을 세울 수 있다.
