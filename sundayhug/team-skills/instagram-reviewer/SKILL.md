# 인스타그램 체험단 후보 발굴 & 검증

RapidAPI Social API4 기반으로 인스타그램 인플루언서를 발굴하고, 체험단 적합성을 종합 평가하는 스킬.

## Prerequisites

- `.env` 파일에 `RAPIDAPI_KEY` 필요
- Node.js 20.6+ (native `--env-file` 지원)
- RapidAPI Social API4 구독: https://rapidapi.com/social-api12-social-api-default/api/social-api4

## API 엔드포인트

Base URL: `https://social-api4.p.rapidapi.com`

### 발굴용
| 엔드포인트 | 파라미터 | 용도 |
|-----------|---------|------|
| `GET /v1/search_users` | `search_query` | 키워드로 유저 검색 |
| `GET /v1/similar_accounts` | `username_or_id_or_url` | 시드 계정 기반 유사 유저 |
| `GET /v1/hashtag` | `hashtag` | 해시태그 게시물에서 유저 추출 |

### 검증용
| 엔드포인트 | 파라미터 | 용도 |
|-----------|---------|------|
| `GET /v1/info` | `username_or_id_or_url` | 프로필 상세 (팔로워, 바이오, 연락처) |
| `GET /v1/posts` | `username_or_id_or_url` | 최근 게시물 분석 |
| `GET /v1/tagged` | `username_or_id_or_url` | 태그된 게시물 (협찬 이력) |
| `GET /v1/reels` | `username_or_id_or_url` | 릴스 콘텐츠 분석 |

## Workflow

사용자 요청에 따라 3가지 모드 중 선택. 복수 모드 조합 가능.

**기본 분석 인원: 최소 20명.** 다수 해시태그 + 유사계정 + 키워드 검색을 병행하여 충분한 후보 풀을 확보한 뒤, 브랜드/비즈니스 계정을 제외한 개인 계정 20명 이상을 상세 분석한다.

```
Task Progress:
- [ ] Step 1: 모드 판별 & 사용자 입력 수집
- [ ] Step 2: 후보 발굴 (모드 A/B) 또는 리스트 수집 (모드 C)
- [ ] Step 3: 후보별 상세 정보 수집
- [ ] Step 4: 자녀 연령대 추정 분석
- [ ] Step 5: 종합 스코어링
- [ ] Step 6: 리포트 생성 & 저장
```

### Step 1: 모드 판별 & 사용자 입력 수집

사용자 요청을 분석하여 모드를 결정한다:

| 요청 패턴 | 모드 | 필요 입력 |
|----------|------|----------|
| "체험단 후보 찾아줘", "인플루언서 발굴" | **A: 해시태그 발굴** | 타겟 해시태그 목록 |
| "비슷한 인플루언서", "이 계정 같은 사람" | **B: 유사 계정 발굴** | 시드 계정명 |
| "이 계정들 검증해줘", "후보 리스트 확인" | **C: 후보 검증** | 유저명 리스트 |

추가로 수집할 정보:
- **타겟 제품/연령대**: 어떤 제품의 체험단인지, 타겟 자녀 연령은 몇 개월~몇 세인지
- **팔로워 기준**: 최소/최대 팔로워 수 (기본: 1,000~50,000)
- **분석 깊이**: 간단(프로필만) vs 상세(게시물+릴스+태그 전체 분석)

### Step 2: 후보 발굴

#### 모드 A: 해시태그 기반 발굴

1. 사용자가 제공한 해시태그 + 자동 추천 해시태그 조합:

```bash
# 해시태그별 게시물 수집
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs hashtag --tag "육아템" --output /tmp/hashtag_result.json
```

2. 결과에서 유저명 추출 (중복 제거)
3. 추천 해시태그 예시 (유아용품):
   - 일반: #육아템, #육아맘, #아기용품, #육아필수템, #아기용품추천
   - 카테고리별: #아기침대, #아기이불, #아기옷, #신생아용품
   - 활동성: #육아일기, #아기성장일기, #맘스타그램, #인스타맘

#### 모드 B: 유사 계정 발굴

```bash
# 시드 계정에서 유사 유저 찾기
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs similar --username "seed_account" --output /tmp/similar_result.json

# 키워드 검색으로 보완
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs search-users --query "육아맘" --output /tmp/search_result.json
```

#### 모드 C: 후보 리스트 검증

사용자가 직접 유저명 리스트를 제공하면 Step 3으로 바로 진행.

### Step 3: 후보별 상세 정보 수집

각 후보에 대해 API 호출. **분석 깊이에 따라 호출 범위 조절**.

```bash
# 프로필 상세 (필수 - 모든 분석에 사용)
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs info --username "TARGET" --output /tmp/info_TARGET.json

# 최근 게시물 (상세 분석 시)
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs posts --username "TARGET" --output /tmp/posts_TARGET.json

# 태그된 게시물 (상세 분석 시)
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs tagged --username "TARGET" --output /tmp/tagged_TARGET.json

# 릴스 (상세 분석 시)
node --env-file=.env ${SKILL_DIR}/scripts/instagram-api.mjs reels --username "TARGET" --output /tmp/reels_TARGET.json
```

**중요**: API 호출 간 1초 간격 유지 (레이트 리밋 방지).

### Step 4: 자녀 연령대 추정 분석

게시물 캡션, 해시태그, 바이오를 분석하여 인플루언서 자녀의 추정 연령대를 파악한다.

#### 연령대 분류 코드

| 코드 | 연령대 | 판별 키워드 |
|------|--------|-----------|
| `NB` | 신생아 (0~3개월) | 신생아, 출산, 산후조리, 배냇저고리, 태열, 수유, 모유, 분유, 젖병, 공갈젖꼭지 |
| `IN1` | 영아 초기 (4~6개월) | 이유식시작, 뒤집기, 100일, 백일, 백일상, 이앓이, 4개월, 5개월, 6개월 |
| `IN2` | 영아 후기 (7~12개월) | 돌잔치, 돌아기, 기어다니기, 이유식, 첫걸음, 잡고서기, 7개월, 8개월, 9개월, 10개월, 11개월, 12개월 |
| `TO1` | 유아 초기 (13~24개월) | 걸음마, 옹알이, 두돌, 13개월, 14개월~24개월, 돌지남 |
| `TO2` | 유아 중기 (25~36개월) | 세돌, 어린이집, 배변훈련, 25개월~36개월, 두돌지남 |
| `PR` | 유아 후기 (37~60개월) | 유치원, 한글공부, 자전거, 37개월~60개월 |
| `KD` | 아동 (5세+) | 초등학교, 학원, 키즈카페, 태권도 |
| `?` | 추정 불가 | 관련 키워드 부족 |

#### 분석 방법

1. **바이오 분석**: "OO맘", "OO개월 아기", "OO년생" 등 직접 언급 확인
2. **캡션 분석**: 최근 게시물 캡션에서 연령 관련 키워드 빈도 카운트
3. **해시태그 분석**: 게시물 해시태그에서 연령 키워드 매칭
4. **가장 많이 매칭된 연령대 코드를 최종 추정값으로 선택**
5. **근거 텍스트 기록**: 판별에 사용된 주요 키워드/문구 3개까지 기록

#### 타겟 연령 매칭

사용자가 제품 타겟 연령을 지정한 경우:
- 정확히 일치: 15점 (만점)
- 인접 연령대 (±1단계): 10점
- 2단계 이상 차이: 5점
- 추정 불가 (?): 7점 (감점하되 탈락시키지 않음)

### Step 5: 종합 스코어링 (100점 만점)

각 후보에 대해 아래 8개 항목을 평가한다.

| 항목 | 배점 | 기준 |
|------|------|------|
| **팔로워 규모** | 10점 | 1K~5K: 7점, 5K~15K: 10점, 15K~50K: 8점, 50K+: 5점, <1K: 3점 |
| **참여율** | 20점 | 5%+: 20점, 3~5%: 16점, 1~3%: 10점, <1%: 4점 |
| **팔로워/팔로잉 비율** | 10점 | 팔로워 > 팔로잉×2: 10점, > 팔로잉: 7점, < 팔로잉: 3점 |
| **콘텐츠 품질** | 10점 | 주 2회+ 포스팅: 10점, 주 1회: 7점, 월 2회 이하: 3점. 캡션 200자+ 가산 |
| **카테고리 적합성** | 15점 | 바이오+게시물에 육아/라이프스타일 키워드 5개+: 15점, 3~4개: 10점, 1~2개: 5점 |
| **자녀 연령 매칭** | 15점 | 타겟 일치: 15점, 인접: 10점, 2단계+: 5점, 불명: 7점 |
| **협찬 이력** | 10점 | 적절한 협찬 경험(3~10건): 10점, 1~2건: 7점, 없음: 5점, 과다(20+): 3점 |
| **프로필 완성도** | 10점 | 사진+바이오+연락처+링크 모두: 10점, 3개: 7점, 2개: 5점, 1개 이하: 2점 |

#### 참여율 계산

```
engagement_rate = (avg_likes + avg_comments) / followers * 100
```

- `avg_likes`: 최근 12개 게시물의 평균 좋아요 수
- `avg_comments`: 최근 12개 게시물의 평균 댓글 수

#### 등급 체계

| 등급 | 점수 | 의미 |
|------|------|------|
| **S** | 90~100 | 즉시 섭외 추천 |
| **A** | 75~89 | 우선 섭외 대상 |
| **B** | 60~74 | 검토 후 섭외 |
| **C** | 40~59 | 조건부 고려 |
| **D** | 0~39 | 부적합 |

#### 카테고리 키워드 사전 (유아용품)

```
육아 키워드: 육아, 육아맘, 맘스타그램, 인스타맘, 아기맘, 워킹맘, 전업맘
제품 키워드: 아기용품, 유아용품, 육아템, 육아필수템, 아기옷, 아기침대, 아기이불
활동 키워드: 이유식, 육아일기, 아기성장, 성장일기, 육아브이로그
라이프스타일: 일상, 홈카페, 집스타그램, 신혼, 살림, 인테리어
```

## Error Handling

| 에러 | 해결 |
|------|------|
| `RAPIDAPI_KEY not found` | `.env`에 `RAPIDAPI_KEY=키` 추가 |
| `429 Rate Limited` | 스크립트가 자동 재시도 (최대 3회). 계속 실패 시 1분 대기 |
| `404 User not found` | 유저명 확인, 비공개 계정일 수 있음 |
| `Empty response` | API 서버 일시 장애. 잠시 후 재시도 |
| 비공개 계정 | 공개 정보만 수집 가능. 리포트에 "비공개" 표시 |

## Tips

- **시드 계정 선택**: 이전에 성공적이었던 체험단 인플루언서를 시드로 사용하면 유사한 품질의 후보를 찾을 확률이 높음
- **해시태그 조합**: 너무 큰 해시태그(#육아) 보다는 중소 해시태그(#육아템추천, #아기침대추천)가 더 타겟팅된 결과를 줌
- **계절 고려**: 시즌성 제품은 관련 시즌 해시태그 추가 (예: #겨울육아, #여름아기옷)
- **반복 사용**: 정기적으로 새 후보를 발굴하고 기존 리포트와 비교하면 체험단 풀을 꾸준히 확대할 수 있음
