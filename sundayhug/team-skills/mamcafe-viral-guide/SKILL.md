# 맘카페 바이럴 댓글 가이드 생성 스킬

제품 정보를 입력하면, **네이버 API 실데이터 기반**으로 맘카페 모니터링 키워드 + 댓글 템플릿을 자동 생성하고, **브라우저에서 바로 편집/저장할 수 있는 인터랙티브 HTML**로 출력한다.

---

## 입력 정보

사용자에게 다음을 확인한다:

1. **제품 정보 소스** (우선순위 순):
   - 로컬 상세페이지 HTML 파일 경로 (가장 정확 — `Read`로 읽기)
   - 제품 URL (공식몰 또는 스마트스토어 — `WebFetch`로 크롤링 시도)
   - 직접 입력 (제품명, 가격, 소재, 사이즈, 주요 특징)
2. **홈페이지 프로모션 정보** (할인, 쿠폰, 세트, 사은품, 프리오더 등)
3. **타겟 맘카페** (기본: 맘스홀릭 베이비)

---

## 실행 프로세스 — 6단계

### STEP 1. 제품 분석

**소스별 수집 방법:**

| 소스 | 방법 | 정확도 |
|---|---|---|
| 로컬 HTML | `Read`로 파일 읽기 → HTML에서 USP/스펙/구성품 추출 | 최상 |
| 제품 URL | `WebFetch`로 크롤링 (JS 렌더링 페이지는 실패 가능) | 중 |
| URL 실패 시 | `search_shop`으로 네이버 쇼핑에서 기본 정보 수집 + `search_blog`로 후기에서 보충 | 보통 |
| 직접 입력 | 사용자에게 입력 요청 | 확실 |

**추출 항목:**
- 제품명, 가격(정가/할인가), 소재, 사이즈, 무게
- 타겟 연령/체중, KC인증, 색상 옵션
- 구성품 목록
- 주요 기능/특징 (모드 전환, 접이식, 통기성 등)

**USP 분석 프레임워크** (6가지 차원에서 도출):
1. **기능적 차별점** — 경쟁사 대비 독특한 기능
2. **안전성/신뢰** — 소재, 인증, 의료 권장 등
3. **편의성** — 착용/설치/세탁/관리
4. **성장 대응** — 사이즈/단계 확장성
5. **감성적 가치** — 부모 불안 해소
6. **시즌/온도 대응** — 계절별 적합성

### STEP 2. 네이버 API 데이터 수집

**모든 키워드에 대해 실데이터를 수집한다. 추정이 아닌 API 실측 값만 사용.**

#### 2-1. 키워드 트렌드 (DataLab API)

`mcp__naver-search__datalab_search` 사용:

```
- startDate: 12개월 전
- endDate: today
- timeUnit: month
- keywordGroups: 5그룹씩 최대 2~3회 호출
```

**필수 조회 그룹:**
- 제품 카테고리 핵심 키워드 (예: "아기침대", "접이식 아기침대")
- 문제/고민 키워드 (예: "분리수면", "수면교육")
- 경쟁 브랜드명 (예: "다이치", "머미쿨쿨")
- 시즌 키워드 (예: "여름 속싸개", "캠핑 아기침대")
- 자사 브랜드명 (예: "썬데이허그") — 인지도 확인

#### 2-2. 키워드별 콘텐츠 수 (블로그 + 카페 API)

**도출된 모니터링 키워드 각각에 대해 블로그/카페 total 건수를 조회한다.**

`mcp__naver-search__search_blog` + `mcp__naver-search__search_cafearticle`:
- display: 1 (건수만 확인하면 되므로)
- 응답의 `total` 값 = 해당 키워드 관련 전체 콘텐츠 수

```
블로그/카페 수 → 경쟁강도 판단 기준:
- 블로그 10만+ 또는 카페 5만+ → 높음 (레드오션)
- 블로그 1만~10만 또는 카페 1만~5만 → 중간
- 블로그 1만 미만 또는 카페 1만 미만 → 낮음 (기회)
- 블로그 100 미만 → 블루오션 (선점 가능)
```

**병렬 호출:** 키워드 10개 이상이므로 블로그/카페를 동시에 병렬 호출하여 속도 최적화.

#### 2-3. 경쟁 제품 분석 (쇼핑 API)

`mcp__naver-search__search_shop` 사용:
- 동일 카테고리 검색 (예: "접이식 아기침대 휴대용")
- display: 10
- 가격, 브랜드, 리뷰 수, 카테고리 추출
- 자사 제품 대비 가격 경쟁력 분석

#### 2-4. 맘카페 실시간 논의 분석

`mcp__naver-search__search_cafearticle` 사용:
- 제품 카테고리 + 고민 키워드로 검색
- sort: date (최신순)
- 실제 엄마들이 쓰는 언어/표현 수집
- 자주 나오는 고민/질문 패턴 파악

### STEP 3. 모니터링 키워드 도출

API 데이터 기반으로 **5개 카테고리**에서 키워드를 도출한다.

| 카테고리 | 설명 | 예시 |
|---|---|---|
| **A. 제품 카테고리** | 제품 유형 직접 관련어 | 아기침대, 접이식 아기침대, 속싸개 |
| **B. 문제/고민** | 부모가 겪는 문제 상황 | 여행 잠자리, 밤에 깨는, 이불 차기 |
| **C. 생애 전환** | 제품 교체/시작 시점 | 바운서 졸업, 분리수면, 수면교육 |
| **D. 경쟁사/대체재** | 경쟁 제품 언급 시 개입 기회 | 다이치, 헬로디노, 머미쿨쿨 |
| **E. 시즌/상황** | 계절/상황별 니즈 | 캠핑, 명절, 여름 |

**각 키워드에 대해 다음 컬럼을 채운다:**

| 컬럼 | 데이터 소스 |
|---|---|
| 키워드 | USP + 카페 분석에서 도출 |
| 트렌드 (ratio) | DataLab API 실측 |
| 블로그 수 | search_blog API total |
| 카페 글 수 | search_cafearticle API total |
| 경쟁강도 | 블로그/카페 수 기반 자동 판정 |
| 진입 상황 | 카페 글 분석에서 도출 |
| 핵심 USP 연결 | 제품 USP와 매칭 |

### STEP 4. 댓글 템플릿 생성

#### 작성 원칙

1. **톤**: 같은 고민을 가진 엄마의 경험 공유. 절대 광고 톤 X
2. **구조**: [공감] → [시도한 것들] → [현재 제품으로 해결] → [구체적 효과]
3. **브랜드 표기**: 초성(ㅅㄷㅇㅎ), "ㅁㅋㅇㅎ", "검색해서 찾았는데" 등 간접 표현
4. **구체성**: "밤에 3번 깨던 애가 1번으로", "30초면 설치 끝" 등 수치/상황 묘사
5. **다양성**: 최소 6개 시나리오 유형에서 총 20개 이상
6. **길이**: 2~4줄 (너무 길면 광고, 너무 짧으면 신뢰도 낮음)

#### 댓글 카테고리 (드롭다운으로 분류)

| 카테고리 | 색상 | 시나리오 |
|---|---|---|
| 여행/외출 | 노랑 | 여행, 친정, 시댁, 캠핑 |
| 코슬립/분리수면 | 보라 | 코슬립, 수면교육, 독립수면 |
| 제품비교 | 파랑 | 경쟁사 비교, 가성비, 추천 |
| 성장전환 | 초록 | 바운서 졸업, 모드 전환, 놀이공간 |
| 안전/통기성 | 회색 | KC인증, 메쉬, 메인침대 |
| 프로모션 | 빨강 | 프리오더, 구매처, 할인 |

---

### STEP 5. 홈페이지 언급내용 정리

맘카페 댓글에서 자연스럽게 언급할 수 있는 구매 관련 정보:

- 프로모션 (할인, 사은품, 프리오더, 세트 등)
- 구매처 (스마트스토어 vs 공식몰 가격 차이)
- 배송 일정
- 액세서리/추가 구매 옵션
- 전문가 추천/인증 정보

### STEP 6. 편집 가능한 인터랙티브 HTML 출력

**최종 결과물은 브라우저에서 바로 편집/저장할 수 있는 HTML 파일로 생성한다.**

#### 파일 위치
```
output/리포트/[제품slug]-mamcafe-guide.html
```

#### HTML 필수 기능

| 기능 | 구현 방법 |
|---|---|
| **셀 편집** | `contenteditable="true"` — 클릭하면 바로 수정 |
| **행 삭제** | 각 행 오른쪽 `x` 버튼 → `confirm()` 후 `remove()` |
| **행 추가** | 각 섹션 헤더에 `+ 행 추가` 버튼 |
| **댓글 삭제** | 각 댓글 `삭제` 버튼 |
| **댓글 추가** | `+ 댓글 추가` 버튼 → 빈 템플릿 생성 |
| **카테고리 변경** | 댓글 `<select>` 드롭다운 |
| **저장** | `Ctrl+S` → `localStorage`에 JSON 저장 |
| **HTML 내보내기** | 현재 편집 상태 그대로 HTML 파일 다운로드 |
| **초기화** | localStorage 삭제 후 새로고침 |

#### HTML 구조

```html
<body>
  <!-- 상단 툴바: 저장/내보내기/초기화 버튼 -->
  <div class="toolbar">...</div>

  <!-- 섹션 1: 홈페이지 관련 언급내용 -->
  <h2>업체별 홈페이지 관련 언급내용 <button>+ 행 추가</button></h2>
  <table id="homepage-table">
    <tr><td contenteditable>항목</td><td contenteditable>내용</td><td class="row-del">x</td></tr>
  </table>

  <!-- 섹션 2: 모니터링 키워드 (API 실데이터) -->
  <h2>모니터링 키워드 <button>+ 행 추가</button></h2>
  <table id="keyword-table">
    <tr>
      <th>키워드</th><th>트렌드</th><th>블로그 수</th>
      <th>카페 글 수</th><th>경쟁강도</th><th>진입 상황</th><th>USP 연결</th>
    </tr>
    <!-- 키워드별 실데이터 행 -->
  </table>

  <!-- 섹션 3: 댓글 예시 -->
  <h2>댓글 예시 <button>+ 댓글 추가</button></h2>
  <div id="comments-container">
    <div class="comment-wrap">
      <div class="comment-header">
        <select>카테고리 선택</select>
        <button>삭제</button>
      </div>
      <div class="comment-body" contenteditable="true">댓글 내용</div>
    </div>
  </div>

  <!-- 섹션 4: 주의사항 -->
  <h2>주의사항</h2>
  <table>절대 금지 / 필수 준수 / 타겟 카페</table>

  <script>
    // localStorage 저장/로드/초기화
    // HTML 내보내기 (Blob 다운로드)
    // 행/댓글 추가/삭제
    // Ctrl+S 단축키
  </script>
</body>
```

#### 경쟁강도 표시 스타일

```html
<span class="lvl lvl-high">높음</span>    <!-- 빨강 배경 -->
<span class="lvl lvl-mid">중간</span>     <!-- 노랑 배경 -->
<span class="lvl lvl-low">낮음</span>     <!-- 초록 배경 -->
<span class="lvl lvl-blue">블루오션</span> <!-- 파랑 배경 -->
```

```
사용자: "ABC 아기침대 맘카페 가이드 만들어줘"

→ STEP 1: 제품 USP 6가지 추출
→ STEP 2: DataLab 트렌드 2~3회 + 블로그 10건+ + 카페 10건+ 병렬 API 호출
→ STEP 3: 모니터링 키워드 15~20개 + 실데이터 채움

→ STEP 5: 홈페이지 언급내용 정리
→ STEP 6: 편집 가능한 HTML 출력 → output/리포트/abc-crib-mamcafe-guide.html
```

## 주의사항

- 맘카페 댓글은 **절대 광고처럼 보이면 안 된다**
- 브랜드명은 자연스러운 방식으로 표기 (초성, 간접 표현, 검색 유도)
- 실제 체험단/실사용자를 통해서만 활동 (브랜드 계정 직접 활동 금지)
- 경쟁사 직접 비방 금지 ("저한테는 이게 더 맞았어요" 정도)
- 같은 카페에 유사한 톤의 댓글 반복 금지 (도배 감지 위험)
- 댓글 예시는 참고 템플릿이며, 실제 사용 시 상황에 맞게 반드시 변형
- **모든 키워드 데이터는 네이버 API 실측값만 사용** — 추정치 사용 금지
